# 🧠DevOps & Infrastructure Engineering Knowledge Base

Welcome to my personal, production-grade knowledge base. This repository is a structured compilation of deep technical notes, manuals, and hands-on guides collected over my 5+ years of infrastructure experience, including enterprise workflows.

## 🗂️ Repository Structure

* **`0.scnc` (Science)** — Computer Science fundamentals, core networking concepts (TCP/IP, DNS, OSI model), and systems theory.
* **`1.os` (Operating Systems)** — Deep-dive Linux administration (Debian, Arch, RHEL internals), kernel tuning, storage management, and Bash scripting.
* **`2.contnr.orchstrtn` (Containers & Orchestration)** — Containerization mechanics. Production Docker setups, multi-stage builds, and Kubernetes (k3s/minikube) architecture, including Ingress (Nginx), Secrets, and microservice deployments.
* **`3.iac` (Infrastructure as Code)** — Configuration management and automation. Enterprise-level Ansible playbooks (IaaS orchestration) and declarative Terraform setups for cloud provisioning.
* **`4.cloud` (Cloud Providers)** — Public and hybrid cloud management, focusing heavily on Yandex Cloud (YC VM provisioning, Object Storage buckets, Cloud-Init configs).
* **`5.git` (Version Control)** — Advanced Git workflows, branching strategies, and repository management best practices.
