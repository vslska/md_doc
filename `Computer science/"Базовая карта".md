
## Языки программирования: два пути исполнения

| Тип                  | Как работает                                                                                  | Плюсы                                                                                                | Минусы                                                                   | Примеры                       |
| -------------------- | --------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------ | ----------------------------- |
| **Компилируемые**    | Исходный код → **компилятор** → машинный код (нативный бинарник `.exe`, `.so`)                | 🚀 Быстро, эффективно по памяти/процессору<br>🛡️ Сложнее реверсить                                  | ⏳ Долгая сборка<br>🌍 Платформозависимость (x86 vs ARM)                  | C, C++, Rust, Go              |
| **Интерпретируемые** | Исходный код → **интерпретатор** → выполняется _построчно_ (часто с промежуточным байт-кодом) | 🐢 Быстрый старт (REPL!)<br>🔄 Кроссплатформенность (главное — интерпретатор)<br>🔧 Легче отлаживать | 🐢 Медленнее (особенно CPU-heavy)<br>📦 Требуется интерпретатор на хосте | Python, JavaScript, Ruby, PHP |
> **Нюанс**: граница размыта ( на примере Python).  
> — Python → компилируется в **байт-код** (`.pyc`), который _потом_ интерпретирует **виртуальная машина** (CPython VM).  
> — JIT-компиляторы (PyPy, V8 в JS) → компилируют «горячие» участки кода _на лету_ в машинный код → гибрид!  
> → Поэтому точнее говорить: **стратегия исполнения**, а не «язык компилируемый/интерпретируемый».

## Фундаментальные понятия
| Понятие                    | Определение                                                                                           | Аналогия                                                                                 | Зачем знать                                                                                            | Частая ошибка                                                                                             |
| -------------------------- | ----------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------ | --------------------------------------------------------------------------------------------------------- |
| **Алгоритм**               | Чёткая, конечная последовательность шагов, решающая задачу при любых допустимых входных данных        | Рецепт торта: «взбить 3 яйца → добавить 200 г сахара → 180°C, 45 мин»                    | Без алгоритма — хаос. Даже `print("Hello")` — алгоритм из 1 шага                                       | Думать, что «алгоритм = сложная математика» — нет: ветвление, цикл, поиск максимума — всё алгоритмы       |
| **Синтаксис**              | Набор правил, задающий _форму_ корректных конструкций языка («грамматика» кода)                       | Пунктуация в предложении: `if x > 0:` — двоеточие _обязательно_                          | Нарушение → SyntaxError. Интерпретатор/компилятор «не поймёт»                                          | Смешивать отступы (пробелы + табы) в Python → IndentationError                                            |
| **Переменная**             | **Имя**, ссылающееся на объект в памяти. Не «коробка», а «ярлык на объекте»                           | Клеймо на животном: «Маша» → корова. Можно переклеить: `Маша = новая_корова`             | Без имён — не запомнить данные. Позволяет абстрагироваться от адресов памяти                           | `a = [1]; b = a; b.append(2)` → `a == [1, 2]`! (изменяемые объекты)                                       |
| **Функция**                | Именованный блок кода, инкапсулирующий логику: принимает аргументы → выполняет → возвращает результат | Автомат по продаже кофе: вбросил монетку (аргумент) → ждёшь → получил стакан (результат) | Повторное использование, тестируемость, уменьшение сложности                                           | Забыть `return` → функция вернёт `None`, даже если внутри что-то посчитала                                |
| **Оператор**               | Символ или ключевое слово, выполняющее действие над операндами (данными)                              | Рычаг на станке: `+` — сложение, `==` — сравнение, `and` — логика                        | Без операторов — нет вычислений. Делятся на: арифметические, сравнения, логические, присваивания и др. | `=` vs `==`: `if x = 5:` → SyntaxError; `x = y == z` → `x = (y == z)`                                     |
| **Рефакторинг**            | Улучшение **внутреннего качества** кода _без изменения его поведения_ (тесты не ломаются!)            | Ремонт в доме: перекладка проводки, утепление, но кухня остаётся на месте                | Код живёт годами — без рефакторинга он «стареет», становится хрупким                                   | «Сначала доделаю фичу, потом рефакторить» → потом «некогда» — рефакторь _малыми шагами_                   |
| **Троттлинг** (Throttling) | **Сознательное ограничение** скорости выполнения (запросов, задач, вызовов)                           | Водяной кран: не даёшь потоку быть слишком сильным, чтобы не прорвало трубу              | Защита от перегрузки: API, БД, внешние сервисы. Часто с sliding window или leaky bucket                | Путать с _rate limiting_ (ограничение клиентов) — троттлинг — внутри системы, rate limiting — на границе  |
| **Идемпотентность**        | Свойство: повторный вызов → **тот же результат + те же побочные эффекты**, что и первый               | Переключатель света: сколько ни жми «вкл», лампа остаётся включённой                     | Критично для отказоустойчивости: повтор запроса при таймауте — безопасен                               | `POST /orders` — не идемпотентен (создаёт новый заказ); `PUT /orders/123` — идемпотентен (перезаписывает) |

## Бонус: как это живёт в Python
| Понятие             | В Python                                                                                   |
| ------------------- | ------------------------------------------------------------------------------------------ |
| **Интерпретатор**   | `python` (CPython) — компилирует `.py` → байт-код → VM исполняет                           |
| **Переменная**      | Всё — объекты. `x = 5` → `x` ссылается на объект `int(5)`                                  |
| **Функция**         | `def`, `lambda`, built-ins (`len`, `map`)                                                  |
| **Операторы**       | `+`, `-`, `and`, `or`, `in`, `is`, `:=` (walrus), `@` (decorators!)                        |
| **Рефакторинг**     | Выделить повторяющийся код → функция; длинный if → полиморфизм; глобалки → класс           |
| **Идемпотентность** | `dict.setdefault()`, `set.add()`, `list.sort()` — идемпотентны по эффекту                  |
| **Троттлинг**       | `time.sleep()`, `asyncio.Semaphore`, `celery` rate limits, `tenacity` retries with backoff |
