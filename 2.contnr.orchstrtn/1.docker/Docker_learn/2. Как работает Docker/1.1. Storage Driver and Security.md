## Storage Driver + Security: OverlayFS, Seccomp, Capabilities

> [!SUMMARY] Суть
> 
> - **OverlayFS** — экономия места за счёт слоёв (общие файлы не дублируются).
> - **Seccomp** — фильтр системных вызовов (контейнер не может вызвать `reboot`, `mount` и т.д.).
> - **Capabilities** — дробление прав root на мелкие «возможности» (не давать всё сразу).

### Union File Systems: как работают слои образа
| Драйвер       | Статус                 | Когда использовать                         |
| ------------- | ---------------------- | ------------------------------------------ |
| **overlay2**  | Рекомендуется (дефолт) | Современный, стабильный, быстрый           |
| **overlay**   | Устарел                | Только для совместимости со старыми ядрами |
| **aufs**      | Не рекомендуется       | Легаси, почти не используется              |
| **zfs/btrfs** | Опционально            | Если вся инфраструктура уже на ZFS/Btrfs   |
**Как проверить свой драйвер:**
```bash
docker info | grep "Storage Driver"
# Вывод: Storage Driver: overlay2
```

**Как работают слои (визуально):**
```bash
Image: myapp:1.0
├── [READ-ONLY] base layer (alpine)
├── [READ-ONLY] pip install dependencies
├── [READ-ONLY] COPY app code
└── [READ-WRITE]  container layer (runtime changes)
```

> [!TIP] Практические следствия
>
>- Каждый `RUN`, `COPY`, `ADD` в Dockerfile = новый слой.
>- Слои кэшируются → порядок команд влияет на скорость сборки.
>- Контейнер добавляет **тонкий writable-слой** поверх образа → изменения не портят оригинал.

**Полезные команды:**
```bash
# Посмотреть слои образа
docker history myapp:1.0

# Посмотреть, какие слои занимают место
docker image inspect myapp:1.0 --format '{{json .RootFS.Layers}}'

# Очистить неиспользуемые слои и кэш
docker builder prune -a
```

### Seccomp or Secure Computing Mode: фильтр системных вызовов

**Задача:** Запретить контейнеру вызывать опасные системные вызовы ядра.

| Профиль                            | Что делает                                                       | Когда использовать                                    |
| ---------------------------------- | ---------------------------------------------------------------- | ----------------------------------------------------- |
| **default** (включён по умолчанию) | Блокирует ~44 вызова (`reboot`, `mount`, `ptrace`, `kexec_load`) | Для большинства приложений — оставляй как есть        |
| **unconfined**                     | Отключает фильтрацию (`--security-opt seccomp=unconfined`)       | Только для отладки или специфичных системных утилит   |
| **custom.json**                    | Твой набор разрешённых/запрещённых вызовов                       | Для hardened-окружений, когда нужен минимальный набор |
**Пример: запустить с кастомным профилем**
```bash
docker run --security-opt seccomp=./my-profile.json myapp
```

**Пример минимального профиля (разрешить только чтение ФС):**
```bash
{
  "defaultAction": "SCMP_ACT_ERRNO",
  "syscalls": [
    {
      "names": ["read", "write", "open", "close", "exit", "exit_group"],
      "action": "SCMP_ACT_ALLOW"
    }
  ]
}
```

Как посмотреть, какой профиль применяется
```bash
docker inspect <container> | grep -i seccomp
# Вывод: "SeccompProfile": "default"
```

[Seccomp](https://docs.docker.com/engine/security/seccomp/?spm=a2ty_o01.29997173.0.0.7ef45171ahb9zS)

### Linux Capabilities: дробление прав root

**Проблема:** `root` в контейнере ≠ `root` на хосте, но по умолчанию у контейнера много лишних прав.

**Решение:** Capabilities разбивают суперпользователя на ~40 отдельных «возможностей».

| Capability           | Что разрешает                      | Пример использования                               |
| -------------------- | ---------------------------------- | -------------------------------------------------- |
| **NET_BIND_SERVICE** | Биндить порты < 1024               | Веб-сервер на порту 80 без полного root            |
| **CHOWN**            | Менять владельца файлов            | Приложение, которое меняет права на файлы в volume |
| **SYS_TIME**         | Менять системное время             | Опасно, обычно не нужно                            |
| **SYS_ADMIN**        | Почти всё (почти root)             | Избегать, если не требуется для специфичных задач  |
| **NET_ADMIN**        | Управлять сетью (iptables, routes) | Контейнер-роутер, VPN, CNI-плагины                 |
| **SYS_BOOT**         | Перезагрузка хоста                 |                                                    |
**Управление через CLI:**
```bash
# Добавить одну возможность
docker run --cap-add=NET_BIND_SERVICE nginx

# Убрать все, добавить только нужные (принцип минимальных привилегий)
docker run --cap-drop=ALL --cap-add=CHOWN --cap-add=SETUID myapp

# Запустить с полным root (не рекомендуется!)
docker run --privileged myapp
```
 
>[!WARNING] `--privileged` — это не просто «все capabilities» 
>Это отключает: seccomp, AppArmor, SELinux, даёт доступ ко всем устройствам (`/dev/*`). Используй только для отладки или системных контейнеров (Docker-in-Docker).

**Полезные команды:**
```bash
# Посмотреть, какие capabilities у процесса внутри контейнера
docker exec <container> cat /proc/self/status | grep Cap

# В десятичном виде → перевести в hex → расшифровать:
# https://man7.org/linux/man-pages/man7/capabilities.7.html
```
