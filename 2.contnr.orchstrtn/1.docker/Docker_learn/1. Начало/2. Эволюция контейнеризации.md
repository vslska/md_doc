> [!SUMMARY] Главная идея 
> Контейнеры — это не новая технология, а **комбинация старых механизмов ядра** (namespaces + cgroups), упакованная в удобный инструмент.

---
## Таймлайн развития
    1979 : chroot : изоляция ФС процесса
    2000 : FreeBSD Jail : + изоляция сети, пользователей
    2001 : Linux-VServer : аналог Jail для Linux
    2006 : Google cgroups : контроль ресурсов (CPU, RAM, IO)
    2008 : LXC : namespaces + cgroups «из коробки»
    2013 : Docker : удобный CLI + образы + реестр

## Что добавила каждая технология
|Технология|Год|Что дала|Чего не хватало|
|---|---|---|---|
|**chroot**|1979|Изоляция корневой директории процесса|Нет изоляции сети, процессов, ресурсов|
|**FreeBSD Jail**|2000|+ изоляция сети, пользователей, хостнейма|Нет контроля ресурсов (до FreeBSD 9)|
|**Linux-VServer**|2001|Port Jail на Linux, общие файлы через hardlinks|Сложная настройка, патчи ядра|
|**cgroups** (Google)|2006|**Контроль ресурсов**: лимиты CPU, RAM, IO|Только ресурсы, без изоляции ФС/сети|
|**namespaces** (Linux)|2002-08|**Изоляция**: процессы, сеть, пользователи, ФС|Только изоляция, без лимитов ресурсов|
|**LXC**|2008|namespaces + cgroups «из коробки»|Сложный CLI, нет удобного формата образов|
|**Docker**|2013|**Удобство**: Dockerfile, образы, реестр, простой CLI|Изначально зависел от LXC (потом свой runtime)|

> [!TIP] Формула современного контейнера
>  **Container = namespaces (изоляция) + cgroups (лимиты) + union FS (слои образа) + удобный CLI**

## Что важно запомнить

1. **Контейнеры ≠ виртуализация**
    - Нет эмуляции железа, нет второго ядра → меньше оверхеда.
    - Но: уязвимость ядра = риск для всех контейнеров.
2. **Docker победил не технологией, а UX**
    - LXC был раньше, но сложный.
    - Docker дал: `Dockerfile` + `docker build` + Docker Hub → стандартизировал процесс.
3. **cgroups и namespaces — это база**
    - Даже если не используешь Docker напрямую (например, Podman, containerd), они работают на тех же механизмах ядра.
    - Полезно знать для отладки: `lsns`, `cgexec`, `/proc/<pid>/cgroup`.
4. **История повторяется**
    - chroot → Jail → LXC → Docker → Kubernetes.
    - Каждый шаг добавлял **удобства** или **безопасности**, но база (изоляция ядра) не менялась.
