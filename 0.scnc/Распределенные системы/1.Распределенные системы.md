# 🌐 Распределенные системы (MOC)

**Распределенная система** — это коллекция независимых вычислительных узлов (серверов), которые взаимодействуют через сеть и координируют свои действия так, чтобы для конечного пользователя выглядеть как единая целостная система.

---

## 🧭 Фундаментальные концепции и теоремы
*   `[[Теорема CAP]]` (Consistency, Availability, Partition Tolerance) — невозможность одновременно обеспечить согласованность, доступность и устойчивость к разделению.
*   `[[PACELC теорема]]` — расширение теоремы CAP, описывающее компромиссы в штатном режиме работы.
*   `[[Задача византийских генералов]]` — проблема достижения согласия в сети с потенциально вредоносными узлами.
*   `[[Модели согласованности данных]]` — Строгая (Strict), Линеаризуемость (Linearizability), Последовательная (Sequential), Итоговая (Eventual Consistency).

---

## 🤝 Алгоритмы консенсуса (Consensus Protocols)
Центральные протоколы, обеспечивающие единое состояние данных на разных машинах:

### Традиционные (Crash Fault Tolerant — CFT)
*   `[[Алгоритм консенсуса Paxos]]` — академический фундамент распределенного согласия.
*   `[[Алгоритм консенсуса Raft]]` — понятный, линейный алгоритм на базе сильного лидера.
*   `[[ZAB]]` (ZooKeeper Atomic Broadcast) — протокол координации для высоконагруженных систем чтения.

### Децентрализованные (Byzantine Fault Tolerant — BFT)
*   `[[Другие алгоритмы консенсуса]]` — обзор PBFT, Proof-of-Work (PoW) и Proof-of-Stake (PoS) для сетей без доверия.

## 🛠 Компоненты инфраструктуры и инструменты
Примеры реальных систем, реализующих эти концепции:
*   `[[etcd]]` — отказоустойчивое хранилище key-value для конфигураций (сердце Kubernetes, использует Raft).
*   `[[Apache ZooKeeper]]` — сервис координации и распределенной блокировки (использует ZAB).
*   `[[Consul]]` — инструмент для Service Discovery и Service Mesh от HashiCorp (использует Raft).
*   `[[ClickHouse]]` / `[[Kafka]]` — системы, активно использующие координацию узлов для репликации данных.

## ⏰ Время и порядок событий
*   `[[Логические часы Лампорта]]` — определение причинно-следственной связи событий без привязки к реальному времени.
*   `[[Векторные часы]]` (Vector Clocks) — продвинутый способ обнаружения конфликтов версий данных.
*   `[[TrueTime API]]` — использование атомных часов и GPS (как в Google Spanner) для синхронизации физического времени.
