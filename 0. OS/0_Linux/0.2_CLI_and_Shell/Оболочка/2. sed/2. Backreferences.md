## Сопоставление подвыражений в `sed` (Backreferences)

**Подвыражения** (или «обратные ссылки») позволяют `sed` не просто находить текст, но и запоминать его части, чтобы затем менять их местами, удалять или дублировать.

### 1. Основы синтаксиса

Чтобы **создать** подвыражение, часть регулярного выражения заключается в экранированные круглые скобки: `\( ... \)`.
- Первая пара скобок создает группу `\1`.
- Вторая — `\2`, и так далее (до `\9`).
---
### 2. Простой пример: перенос цифры

**Задача:** Изменить `image.jpg.1` на `image1.jpg`.

1. **Шаблон поиска:** `image\.jpg\.\([1-3]\)`
> - `image\.jpg\.` — ищет точное совпадение текста (точка экранируется, чтобы не значить «любой символ»).
 > - `\([1-3]\)` — находит цифру от 1 до 3 и **сохраняет** её в ячейку памяти №1.
2. **Шаблон замены:** `image\1.jpg`
> -  `\1` — вставляет то, что было найдено в первых скобках.

**Команда**:
```bash
$ ls | sed "s/image\.jpg\.\([1-3]\)/image\1.jpg/"
```
---
### 3. Сложный пример: изменение структуры имени

**Задача:** Изменить `apple.jpg.1` на `apple1.jpg`.

Здесь нам нужно разбить строку на три переменные части:

1. **Имя (слово):** `\([a-z][a-z]*\)` — сохраняется в `\1`.
2. **Расширение (3 буквы):** `\([a-z][a-z][a-z]\)` — сохраняется в `\2`.
3. **Номер (цифра):** `\([0-9]\)` — сохраняется в `\3`.
---
### Сборка команды

Между группами мы ставим экранированные точки `\.`, которые разделяют их в исходном имени файла.

**Полная конструкция:**

- **Найти:** `\([a-z][a-z]*\)\.\([a-z][a-z][a-z]\)\.\([0-9]\)`
- **Заменить на:** `\1\3.\2` (Имя + Цифра + Точка + Расширение)

**Команда:**

```bash
$ ls | sed "s/\([a-z][a-z]*\)\.\([a-z][a-z][a-z]\)\.\([0-9]\)/\1\3.\2/"
```

**Результат:**

- `apple.jpg.1` → `apple1.jpg`
- `banana.png.2` → `banana2.png`
---
### 4. Как это применить для реального переименования?

`sed` сам по себе только печатает текст в консоль. Чтобы переименовать файлы, результат работы `sed` обычно передают в оболочку (shell).

Один из способов — использование цикла или утилиты `rename`, но классический метод через `sed` выглядит так (генерация команд `mv`):
```bash
ls *.jpg.* | sed -E 's/([a-z]+)\.([a-z]+)\.([0-9]+)/mv & \1\3.\2/' | sh
```

>_Здесь использован флаг `-E` (расширенные регулярные выражения), который позволяет писать скобки `()` без обратных слэшей для удобства._
