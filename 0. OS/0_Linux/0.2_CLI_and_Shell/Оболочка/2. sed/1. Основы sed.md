
## Основы sed
Команда `sed` преобразует текст из файлов или из стандартного ввода, используя последовательность инструкций, которую называют `sed`-сценарием.

Вызову сценарий
```bash
$ sed script input-files
```
или использую параметр `-e` для поддержки нескольких сценариев, которые последовательно обрабатывают ввод:
```bash
$ sed -e script1 -e script2 -e script3 input-files
```
Также можно хранить `sed`-сценарии в файлах и обращаться к ним с параметром `-f`, тогда они будут запускаться последовательно:
```bash
$ sed -f script-file1 -f script-file2 -f script-file3 input-files
```
Наиболее часто используемым сценарием является подстановка, которая заменяет одни строки другими. 
```bash
s/regexp/replacement/
```
> **regexp** — регулярное выражение для сопоставления с каждой входной строкой.
> **replacement** — строка для замены совпадающего текста.

> [!важн.]
> При вводе `sed`-сценария в командной строке заключай его в кавычки, чтобы оболочка не вычисляла специальные символы `sed. При необходимости используй одинарные или двойные кавычки.


Косая черта  `/` в подстановке может быть *заменена* любым другим удобным
символом. Это полезно, когда само регулярное выражение включает слеш,
который в противном случае пришлось бы экранировать. Следующие три
сценария sed эквивалентны:

```bash
s/one/two/
s_one_two_
s@one@two@
```

Параметр `i` делает совпадения нечувствительными к регистру:
```bash
$ echo Efficient Stuff | sed "s/stuff/linux/" # чувствит. к регистру
$ echo Efficient Stuff | sed "s/stuff/linux/i" # не чувств к регистру
```

`g` (global) заменяет все вхождения регулярного выражения, а не только первое:
```bash
$ echo efficient stuff | sed "s/f/F/" # Заменяет первую букву
$ echo efficient stuff | sed "s/f/F/g"
```

 `sed`-сценарий  **удаление**, удаляющий строки по их номеру:
```bash
seq 1 5 | sed 3d # удаляет  3 строку 
1
2
4
5 
```

Сценарий, **удаляющий** строки, которые соответствуют *регулярному выражению*:
```bash
$  seq 1 10 | sed '/[13579]$/d' # Удаляет строки заканч. на нечетные
2
4
6
8
10
```
