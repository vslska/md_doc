### Лучшие способы вернуться в каталог
Предположим,  находились в отдаленном вложенном каталоге и запустили `cd`, чтобы перейти куда-то еще:
```bash
$ pwd
/home/max/work
$ cd /etc/
```

Но затем, захотели вернуться, в каталог `work`, вводить повторно путь до `home/max/work` не нужно, для этого можно *выполнить*
```bash
$ cd -
/home/max/work
```

Данная команда возвращает обратно в каталог, в котором мы находились.
Чтобы перемещаться `туда-обратно`, можно повтороно выполнить `cd -`
Однако есть одна загвоздка: оболочка запоминает только один предыдущий каталог **за раз.** Чтобы можно было выполнить быстрое перемещение м\у несколькими каталогами, можно использовать `pushd` и `popd`

### Переключение м\у несколькими каталогами: `pushd` and `popd`hj (Стек каталогов)

*Стек каталогов* — это список каталогов, которые посетили в текущем экземпляре оболочки и решили отслеживать.
Управляя стеком , выполняем 2 операции:
*Вталкивание* - pushing добавляет каталог в начало списка.
*Извлечение* - popping удаляет верхний каталог из стека.

Первоначально стек содержит только текущий каталог, но можно добавлять *(вталкивать)* и удалять *(извлекать)* каталоги и **быстро переходить между ними.**

> [!NOTE] Важно
> Каждый экземпляр оболочки поддерживает свой собственный стек каталогов

### `pushd` - помещение каталога в стек.

Команда `pushd` (сокращение от *push directory*) выполняет следующие действия:
1. Добавляет заданный каталог в вершину стека.
2. Выполняет `cd` в этот каталог.
3. Выводит на экран стек сверху вниз для справки.

Создадим стек из 4-х каталогов:
```bash
$ pwd
/home/max
$ pushd /var/lib
/var/lib ~ /etc ~/my/git ~/my/game ~/my ~/my/app
$ pushd ~/.config
~/.config /var/lib ~ /etc ~/my/git ~/my/game ~/my ~/my/app
$ pushd /opt
/opt ~/.config /var/lib ~ /etc ~/my/git ~/my/game ~/my ~/my/app
$ pushd /proc
/proc /opt ~/.config /var/lib ~ /etc ~/my/git ~/my/game ~/my ~/my/app
```
Оболочка печатает стек после каждой команды `pushd`. Текущий каталог — левый (верхний).

####  `dirs` просмотр стека каталогов
С помощью команды `dirs` можно вывести список каталогов:
```bash
$ dirs
/proc /opt ~/.config /var/lib ~ /etc ~/my/git ~/my/game ~/my ~/my/app
# Или с флагом -p, отобразит сверху-вниз
$ dirs -p
/proc
/opt
~/.config
/var/lib
~
/etc
~/my/git
~/my/game
~/my
~/my/app
```

Можно передать вывод команде `nl` для нумерации строк с нуля:
```bash
$ dirs -p | nl -v0
     0  /proc
     1  /opt
     2  ~/.config
     3  /var/lib
     4  ~
     5  /etc
     6  ~/my/git
     7  ~/my/game
     8  ~/my
     9  ~/my/app
```

Но все намного проще, запустив с флагом `-v`  ( *здесь должен trollface, однако у меня сей сейчас интрент закончился, а эмодзи себе не поставил в арчик ))* )
```bash

2       ~/.config

4       ~
5       /etc

```

#### Извлечь каталог из стека
Команда `popd` (pop directory) противоположна команде `pushd`:
1. Удаляет один каталог из вершины стека.
2. Выполняет команду cd в новый верхний каталог.
3. Выводит на экран стек сверху вниз для справки.

 Например в нашем стеке *4 каталога*:
```bash
 $ dirs -v
0       ~/my/key
1       ~/my/obsidian
2       ~/my/game
3       ~/my/other/tets/fff
```

Далее несколько раз запустим `popd`, которая "пройдётся" *по каталогам сверху вниз*:
```bash
➜  key popd
~/my/obsidian ~/my/game ~/my/other/tets/fff ~
➜  obsidian popd
~/my/game ~/my/other/tets/fff ~
➜  game popd
~/my/other/tets/fff ~
➜  fff popd
~
```

Данные команды очень экономят *время*, лучше сделать *алиасы* 

### Поменять местами каталоги
`pushd` без аргументов  меняет местами *2 верхних каталога*
```bash
$ dirs -v
0       ~/my/git
1       ~/my/project
2       /etc/zsh
$ pushd
~/my/project ~/my/git /etc/zsh ~
$ pushd
~/my/git ~/my/project /etc/zsh ~
```

*3-й* каталог остается на месте, так же стоит обратить внимание, что `pushd` ,при переключении между *2-мя каталогами*, подобен `cd`, но не ограничен  запоминанием *одного каталога*.

### Углубление в стек:
Что если мы хотим перейти в *другие каталоги в стеке*, заместо *2-х верхних*:
```bash
$ pushd +N # Cдвигает N каталогов с вершины стека вниз и переходит.
$ pushd -N # Сдвигает в противоположном направлении
```

>`popd` аналогично принимает *данные аргументы*
> Определять позицию каталога можно  с помощью `dirs -v`
