
# Знакомство с ком.оболч.
## Вычисление переменных
Ком.оболочка может определять переменные и сохранять в них значе-
ния.
Для вывода на экран значений переменных `HOME` и `USER` 
вводим `printenv` ( прим. user: `maxim`):
```bash
$ printenv HOME
/home/maxim
$ printenv USER
maxim
```
Чтобы вычислить переменную, ставим `$` .Например, $HOME вернет `home/maxim.`
Как оболочка обрабатывает командную строку, — запустить команду `echo`, которая выводит свои аргументы (после того как болочка завершит их вычисление):
```bash
$ echo My name is $USER and my files are in $HOMEs
My name is maxim and my files are in /home/maxim
```

---
## Откуда берутся значения переменных
Переменные, как `USER` и `HOME`, предопределены оболочкой. Их значения
устанавливаются автоматически при входе в систему.
Можно определить или изменить переменную в любое время: присвоить ей
значение, используя  синтаксис:
```bash
name=value
```
Например, если часто воркаете с каталогом /home/maxim/Projects, можно
присвоить его имя переменной:
```bash
$ work=$HOME/Projects
```
и использовать его как удобное сокращение при работе с cd:
```bash
$ cd $work
$ pwd
/home/maxim/Projects
```
>!Важно:
>*При определении переменной не допускается использование пробелов около
>знака равенства.*

## Переменные и заблуждения

Например команда `echo` не проверяет переменную. 
```bash
$ echo $HOME
/home/maxim
```
Она просто выводит на экран любые аргументы, которые передали.
Значение `$HOME` вычисляет командная оболочка перед запуском команды echo.

С точки зрения `echo`
```bash
$ echo /home/maxim
```

Оболочка вычисляет переменные, шаблоны и другие конструкции
перед выполнением команды.

## Сокращение команд с помощью псевдонимов
В ком. об. также есть имена, которым соответствуют команды. Они
называются псевдонимами (`aliases`). Определим псевдоним, поставив после него
знак равенства и команду:
```bash
$ alias g=grep       # Пример команды без аргументов				
$ alias ll="ls -l"   # Пример команды с аргументами, кавычки необходимы	
```

Можно определить псевдоним с тем же именем, что и у существующей
команды,заменив  её, эта практика называется  затенением (`shadowing`) команды.
Псевдонимы имеют приоритет над командами с тем же именем.

Чтобы вывести все псевдонимы и их значения, которые определены на данный
момент в вашей командной оболочке, запустите `alias` без аргументов:
```bash
$ alias
alias g='grep'
alias ll='ls -l'
```

Чтобы вывести значение конкретного псевдонима, запустите `alias` и передайте
его имя в качестве аргумента:
```bash
$ alias g
alias g='grep'
```

Чтобы удалить псевдоним из оболочки, выполните `unalias`:
```bash
$ unalias g
```

## Окружение и файлы инициализации, краткая версия
Запущенная оболочка хранит в переменных множество важной информации:
путь для поиска, текущий каталог, предпочитаемый вами текстовый редактор,
настраиваемое приветствие командной строки и многое другое. Переменные за-
пущенной командной оболочки в совокупности называются *окружением*. Когда
пользователь выходит из командной оболочки, ее *окружение* уничтожается.
Было бы  утомительно при каждом запуске оболочки определять
ее *окружение* вручную. Поэтому оно определяется в сценариях, называемых
*файлами запуска* и *файлами инициализации*, которые автоматически выполня-
ются при запуске командной оболочки. 
Прим. файла `.bashrc`
```bash
# Задать путь для поиска
PATH=$HOME/bin:/usr/local/bin:/usr/bin:/bin
# Задать приглашение командной строки
PS1='$ '
# Задать предпочитаемый текстовый редактор
EDITOR=emacs
# Начать в моем рабочем каталоге
cd $HOME/Work/Projects
# Определить псевдоним
alias g=grep
# Сердечно поприветствовать
echo "Welcome to Linux, friend!"
```
Изменения, которые вы вносите в $HOME/.bashrc, не влияют на запущенные
экземпляры оболочки. Но вы можете заставить запущенный экземпляр заново
выполнить $HOME/.bashrc с помощью одной из следующих команд:
```bash
$ source $HOME/.bashrc # Используется встроенная команда source
$ . $HOME/.bashrc      # Используется точка
```
Этот процесс известен как считывание файла инициализации. Говоря «использовать исходный файл .bashrc», имеют в виду выполнение одной из предыдущих команд.
