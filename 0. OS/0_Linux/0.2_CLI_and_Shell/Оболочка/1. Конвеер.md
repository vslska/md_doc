## Ввод, вывод и каналы.
Большинство команд Linux считывают ввод с клавиатуры, отображают вывод на экране или делают и то, и другое. В Linux эти операции чтения и записи имеют собственные названия:

`stdin` ( стандартный ввод, или стандартный входной поток)
Поток данных, который Linux считывает при вводе с клавиатуры. Когда
вы вводите любую команду в терминале после приглашения командной
строки, вы передаете данные в стандартный входной поток `stdin`.

`stdout` (стандартный вывод, или стандартный выходной поток)
Поток данных, который Linux выводит на ваш дисплей. Когда вы за-
пускаете команду ls для печати имен файлов, результаты появляются
в стандартном выходном потоке stdout.

Можно подключать стандартный вывод одной комманды к стандартному вводу другой комманды.

Использование каналов - неотъемлемая часть работы с linux
Комманды `wc, head, cut, grep, sort и uniq` для работы с потоком
### wc
Команда `wc` выводит количество строк и символов в файле:
```bash
wc animals.txt
7 50 309 animals.txt
```
Сообщает что в фалйле animals.txt 7 строк, 50 слов и 309 символов.
`wc` также учитывает скрытые символы новой строки.
Опции -l, -w и -c указывают `wc` печатать только количество строк, слов и символов соответственно.

### head
Команда head выводит первые строки файла. Выведем первые три строки файла
```bash
head -n3 animanls.txt

python Programming Python 2010 Lutz Mfkr
snail ssh, the Secure Shell 2005 Barret Daniel
alpaca Intermediate Perl 2012 Schwarz Randal
```
Если вы запрашиваете больше строк, чем содержится в файле, `head` выведет
весь файл (как это делает `cat`). Если вы опустите параметр -n, заголовок по
умолчанию будет состоять из 10 строк

### cut 
Команда cut выводит одну или несколько колонок из файла. Например, выведем
все названия книг, которые расположены во второй колонке файла animals.txt
```bash
cut -f2 animals.txt

Programming Python
ssh, the Secure Shell
Intermediate Perl
MySql HA
Linux in a Nutshell
Cisco IOS in a Nutshell
Writing Word Macros
```
Команда `cut` поддерживает два способа определения, что считать колонкой.
Первый — разделение по полям (`-f`), когда входные данные состоят из строк
(полей), каждая из которых разделена одним символом табуляции.
Также можно определить колонку для команды cut по положению символа
в строке с использованием параметра `-c`
### grep
grep — чрезвычайно мощная команда,  которая выводит строки, соответствующие заданному шаблону.
Например можно вывести строки, содержащие слово 'Nutshell'
```bash
grep Nutshell animals.txt

horse	Linux in a Nutshell	2009	Siever, Ellen
donkey	Cisco IOS in a Nutshell	2005	Boney, James
```
grep читает стандартный ввод и записывает стандартный вывод, что делает эту команду идеальной для конвейеров. Опция -v выводит строки, исключающие данное слово. 

Можно подсчитать строки, скоклько у нас каталогов, например в /usr/lib
```bash
$ ls -l /usr/lib | cut -c1 | grep d | wc -l
```

### sort
Команда sort сортирует строки файла в порядке возрастания (по умолчанию)
```bash
sort animals.txt

alpaca	Intermediate Perl	2012	Schwarz, Randal
donkey	Cisco IOS in a Nutshell	2005	Boney, James
horse	Linux in a Nutshell	2009	Siever, Ellen
oryx	Writing Word Macros	1999	Roman, Steven
python	Programming Python	2010	Lutz, Mfkr
robin	MySql HA	2014	Bell, Charlez
snail	ssh, the Secure Shell	2005	Barret, Daniel
```
или в порядке убывания (с параметром -r):
`sort` может сортировать строки в алфавитном порядке (по умолчанию) или
в числовом порядке (с опцией -n)
```bash
cut -f3 animals.txt | sort -n # по возрастанию, опция -nr, по убыванию

1999
2005
2005
2009
2010
2012
2014
```

Чтобы узнать год выхода самой новой книги в animals.txt, направим вывод sort
на ввод head и напечатаем только первую строку:
```bash
$ cut -f3 animals.txt | sort -nr | head -n1

2014
```

sort и head — мощные партнеры при работе с числовыми данными, если
они расположены по одному в каждой новой строке. Вы можете вывести
максимальное значение с помощью такого конвейера:

Отсортированный спиок всех пользоталей в системе
```bash
cat /etc/passwd | cut -d: -f1 | sort

alpm
avahi
bin
brltty
colord

```

Определяем, есть ли у  пользователя учетная запись в системе
```bash
cut -d: -f1 /etc/passwd | grep -w "brltty"
brltty
```
Параметр -`w`указывает команде `grep` сопоставлять только слова целиком, а неих части.

### uniq

Команда `uniq` обнаруживает повторяющиеся соседние строки в файле.
Поумолчанию она удаляет повторы.
```bash
cat letter
A
A
A
B
B
A
C
C
C
C
uniq letters
A
B
A
C
```
Rоманда `uniq` сократила первые три строки A до одной A, но оставила последнюю A на месте, потому что она не была соседней с первыми тремя
С помощью параметра `-c` можно подсчитать количество повторяющихся строк:
```bash
uniq -c letters
3 A
2 B
1 A
4 C

Часто встречающаяса оценка
cut -f1 grades | sort | uniq -c | sort -nr | head -n1 | cut -c9
B
```

## Обнаружение дубликатов файлов
Предположим,мы находимся в каталоге, который забит файлами JPEG, и хотим узнать, не дублируются ли они:
```bash
ls
image001.jpg image005.jpg image009.jpg image013.jpg image017.jpg
image002.jpg image006.jpg image010.jpg image014.jpg image018.jpg
```
понадобится ещеодна команда, md5sum, которая проверяет содержимое файла и вычисляет контрольную сумму:
```bash
$ md5sum *.jpg
146b163929b6533f02e91bdf21cb9563 image001.jpg
⋮
```
Вы-числим все контрольные суммы, с помощью `cut` изолируем первые 32 символа каждой строки и отсортируем строки, чтобы дубликаты располагались рядом друг с другом, добавим команду `uniq` для подсчета повторяющихся строк
```bash
md5sum *.jpg | cut -c1-32 | sort | uniq -c
1 1258012d57050ef6005739d0e6f6a257
2 146b163929b6533f02e91bdf21cb9563
1 17f339ed03733f402f74cf386209aeb3
⋮
```
Отсортируем результаты по убыванию, и значения, которые
больше 1, будут выведены в верхней части списка:
```bash
md5sum *.jpg | cut -c1-32 | sort | uniq -c | sort -nr
3 f6464ed766daca87ba407aede21c8fcc
2 c7978522c58425f6af3f095ef1de1cd5
2 146b163929b6533f02e91bdf21cb9563
1 d8ad913044a51408ec1ed8a204ea9502

```
Удалим все недублированные элементы. Их контрольным суммам предшествуют **шесть** пробелов, число *1* и **один** пробел.Используем `grep -v`для удаления этих строк:
```bash
md5sum *.jpg | cut -c1-32 | sort | uniq -c | sort -nr | grep -v "      1 "
3 f6464ed766daca87ba407aede21c8fcc
2 c7978522c58425f6af3f095ef1de1cd5
2 146b163929b6533f02e91bdf21cb9563
```
Идентифиц ируем файлы с заданной контрольной суммой,
выполнив поиск с помощью `grep` и почистим вывод с помощью `cut`:
```bash
md5sum *.jpg | grep 146b163929b6533f02e91bdf21cb9563 | cut -c35-

image001.jpg
image003.jpg
```
