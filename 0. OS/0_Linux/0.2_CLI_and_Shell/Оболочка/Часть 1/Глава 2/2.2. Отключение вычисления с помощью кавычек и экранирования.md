## Отключение вычисления с помощью кавычек и экранирования
### Пробел в мени файла
>В linux пробел является **разделителем**,чтобы оболочка рассматривала пробел как **символ** в имени файла  используем *кавычки*  или *backslash*
```bash
$ cat 'Efficient Linux Tips.txt'
$ cat "Efficient Linux Tips.txt"
$ cat Efficient\ Linux\ Tips.txt
```
### `''`
**Одинарные кавычки ( ' ' )**: Полностью отключают интерпретацию всех специальных символов, включая переменные и команды, сохраняя все как есть, буквально
```bash
$ echo 'Variable $HOME' # Выведет: Variable $HOME (отк. интерп. переменной $HOME, пробел сохранен )
```

### `""`
**Двойные кавычки ( " " )**: Сохраняют пробелы, но позволяют интерпретировать переменные (например, `$VAR`), команды в `` ` `` или `$()` и экранировать символы с помощью `\`
```bash
echo "Hello world" # Выведет: Hello world (пробел сохранен)
echo "Hello   world" # Выведет: Hello   world (несколько пробелов сохранены)
echo "Variable is $HOME" # Выведет: Variable is /home/user (переменная расширяется)
echo "\"Hello\"" # Выведет: "Hello" (экранируем кавычку внутри двойных кавычек)

```

## `\`
Обратная косая черта, также называемая *экранирующим символом*, указывает
оболочке воспринимать буквально символ, находящийся после нее. В следующей
команде экранируется знак доллара:
```bash
$ echo \$HOME
$HOME
```
Обратная косая черта действует как экранирующий символ в двойных кавычках:
```bash
$ echo "Variable of \$HOME is $HOME"
Variable of $HOME is /home/smith
```
но не в одинарных:
```bash
$ echo 'Variable of \$HOME is $HOME'
Variable of \$HOME is $HOME
```
Можно использовать обратную косую черту, чтобы экранировать символ двой-
ных кавычек внутри двойных кавычек:
```bash
echo "This message is \"sort of\" interesting"
This message is "sort of" interesting
```
Обратная косая черта в конце строки отключает функцию невидимого символа
новой строки, позволяя командам оболочки занимать несколько строк:
```bash
echo "This is a very long message that needs to extend \
onto multiple lines"
This is a very long message that needs to extend onto multiple lines
```
 Пример читаемого конвеера, как продолжение строки:
```bash
 cut -f1 grades \
| sort \
| uniq -c \
| sort -nr \
| head -n1 \
| cut -c9
```
Начальная обратная косая черта перед псевдонимом экранирует его, заставляя
оболочку искать команду с тем же именем. Затенение при этом игнорируется:
```bash
$ alias less="less -c" # Определение псевдонима
$ less myfile  # Запуск псевдонима, который определен как less -c
$ \less myfile  # Запуск стандартной команды less, а не псевдонима
```
