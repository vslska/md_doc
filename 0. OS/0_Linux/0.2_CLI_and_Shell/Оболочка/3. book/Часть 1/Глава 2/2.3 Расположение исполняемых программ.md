## Расположение исполняемых программ
Предварительно составленный список путей каталогов, хранящиеся в памяти оболочки  ( наз. `путём для поиска исполняемых программ`.),  как значение  в переменной оболочки $PATH
```bash
echo $PATH
/home/maxim/bin:/home/maxim/.local/bin:/usr/local/bin
```
> Каталоги в пути для поиска разделяются двоеточиями (:)
> Можно преобразовать двоеточия в переводы строки, передав вывод команде tr,
> которая заменяет символы
```bash
echo $PATH | tr : "\n"
/home/vslsk/yandex-cloud/bin
/home/vslsk/bin
/home/vslsk/.local/bin
/usr/local/bin
```

> *Некоторые оболочки запоминают (кэшируют) пути к программам по мере их обнаружения,сокращая количество поисковых запросов в будущем.*

Чтобы найти программу в списке каталогов для поиска, используй команду
`which`:
```bash
which cp
/usr/bin/cp
```
или встроенную в оболочку более мощную и информативную команду type, ко-
торая также находит псевдонимы, функции и встроенные в оболочку команды1.
```bash
type cp
cp is /usr/bin/cp
```
Если есть программы с одинаковыми названиями, расположенные в разных ка-
талогах, например /usr/bin/less и /bin/less, оболочка запустит тот файл, который
найдет первым. Можно переопределить команду, поместив соответствующий файл в каталог, который будет просматриваться раньше, например личный каталог $HOME/bin.
> Когда оболочка ищет команду по имени, она прежде всего проверяет,
> является ли это имя псевдонимом. Именно поэтому псевдоним затеняет
> команду (имеет более высокий приоритет) с таким же именем.
