##  История команд
`История команд` — это список команд, которые вы выполняли в интерактивной оболочке. Чтобы просмотреть историю, запустите команду `history`, встроенную в оболочку.
```bash
$ history
 8815  git add .
 8816  git commit -m "fix"
 8817  git push
```

Вывод `history` может состоять из сотен (и более) строк. Ограничьте его самыми последними командами, добавив целочисленный аргумент, который указывает количество выводимых строк:
```bash
$ history 3
  142  cd
  143  exit
  144  history 3
```

Поскольку `history` записывает в стандартный вывод, вы можете обрабатывать ее вывод с помощью `конвейеров`. Например, просмотрите свою историю по одному экрану за раз:
```bash
$ history | less #От самой последней команды к первой запущенной		
$ history | sort -nr | less #От самой первой команды к последней запущенной
```

Или вывести комманды содержащие `ls`
```bash
$ history | grep -w ls
```

Очистка истории
```bash
$ history -c
```

###  Изменение количества команд 
Чтобы посмотреть, какое максимальное количество команд истории используем:
```bash
$ echo $HISTSIZE
50000
```

Можно задать количество команд в  истории:
```bash
$ HISTSIZE=70000 # 70000 строк истории
$ HISTSIZE=-1    # бесконечное количество команд истории
```

Игнорируем добавление команды в историю ( *при условии что одна идёт за другой*)
```bash
$ HISTCONTROL=ignoredups
```

Каждый экземпляр интерактивной оболочки имеет свою независимую от других историю команд.

Всякий раз, когда интерактивная оболочка завершает работу, она записывает свою историю в файл `$HOME/.bash_history` или другой, хранящийся в переменной оболочки `HISTFILE`:
```bash
$ echo $HISTFILE
/home/username/.bash_history
```

Новые экземпляры оболочки считывают при запуске историю файла `$HISTFILE`

Переменная `HISTFILESIZE` определяет, сколько строк истории записывается в файл. Если  изменять `HISTSIZE`, управляя размером истории в памяти, желательно обновить значение `HISTFILESIZE`:
``` bash
$ echo $HISTFILESIZE
500 # Если пусто, то кол-во строк бесконечно
$ HISTFILESIZE=10000
```

### Расширение истории команд
**Расширение истории** — это функция оболочки, которая обращается к истории команд с помощью *специальных выражений*. Эти выражения начинаются с восклицательного знака, который пользователи Linux традиционно называют `bang`. 

!!!!! ОСТОРОЖНО, КОМАНДЫ  ВЫПОЛНЯЮТСЯ СРАЗУ, ЧТОБЫ ИЗБЕЖАТЬ ставим `:p`  ( в `zsh`  подставляет на выполнение, но не выполняет сразу!)

Два восклицательных знака подряд (`bang-bang`) означают предыдущую команду:
```bash
$ !! # bang-bang *выполнить предыдущую команд*
```

Чтобы обратиться к последней команде, начинающейся с `определенного выражения`, поставьте перед ним восклицательный знак.
```
$ !grep
```

Чтобы обратиться к последней команде, которая содержала заданное выражение `где-либо`, а не только в начале строки, окружите его `вопросительными знаками`:
```bash
$ !?grep?
history | grep -w cd
```

Также можно найти команду из истории  по её абсолютной позиции:
```bash
$ history | grep hosts
1203 cat /etc/hosts

$ !1203 # Выполняем команду под номером 1023
cat /etc/hosts
```

Отрицательное значение извлекает команду из истории по ее относительной, а не абсолютной позиции ( сколько  шагов назад была выполнена):
```bash
$ !-2
clear
```

```bash
$ !-2:p # выводим команду 2 шага назад ( не выполняя `:p`)
clear
$ !!    # Выполняем команду
```

Расширение истории иногда называют `bang-командами`, но выражения вроде `!!` и `!grep` не являются `командами`. Это `строковые выражения`, которые можно поместить в `любом месте команды`. Использя команду `echo` для  вывода значения `!!` в `stdout`, не выполняя его, и подсчитаем количество слов с помощью `wc`:
```bash
$ ls -l /etc | head -n3 # Запустим команду

$ echo "!!" | wc -w # Подсчитаем количество слов в пред команде
echo "ls -l /etc | head -n3" | wc -w
6
```

###  Удаление с помощью расширения команд (без ошибочного удаление,, ОТ КРИВОРУКОВТИ) )

1. `Проверка`. Перед запуском `rm` запустить `ls` с `нужным шаблоном`, чтобы
увидеть, какие файлы совпадают:
```bash
ls *.txt
a.txt b.txt c.txt
```

2. `Удаление`. Если вывод `ls` прошел проверку, запустим `rm !$`, чтобы удалить те же файлы, которые были сопоставлены с шаблоном.
```bash
$ rm !$
rm *.txt
```

Расширение истории `!$` означает «*последнее* слово, которое ввели в предыдущей команде»

Похожий метод заключается в просмотре содержимого файла с помощью ко-
манды `head` перед его удалением. Убедившись, что выбрали правильный файл, можно  запустить `rm !$`

Также в оболочке имеется расширение истории `!*`, которое соответствует *всем* аргументам, введенным в предыдущей команде, а не только *последнему*:
```bash
$ ls *.txt *.o *.log
a.txt b.txt c.txt main.o output.log parser.o
$ rm !*
rm *.txt *.o *.log
```

На практике  *лучше ипользовать* `!*` значительно *реже*, чем `!$`, так как есть риск, что в случае ошибки звездочка `*` будет интерпретирована как *символ шаблона* для имен файлов. 

### Инкрементальный поиск по истории команд
Инкрементальный  поиск — это самый простой и быстрый способ вспомнить команды из истории, даже те, которые запускал давно.
- В приглашении оболочки нажимаем Ctrl-R (R означает обратный (reverse) инкрементальный поиск).
- Вводите любую часть ранее запущенной команды — начало, середину или конец.
- С каждым введенным символом оболочка находит ближайшую в истории команду, которая соответствует вашему вводу. 

Вот еще несколько трюков с инкрементальным поиском:
- Чтобы вызвать последнюю строку, которую вы искали и выполняли, начните с нажатия `Ctrl-R` два раза подряд.
- Чтобы остановить инкрементальный поиск и продолжить работу над текущей командой, нажмите Escape, `Ctrl-J` или любую клавишу для редактирования в командной строке ,например клавишу со стрелкой влево или вправо.
- Чтобы выйти из инкрементального поиска и очистить командную строку, нажмите `Ctrl-G` или `Ctrl-C`.

## Редактирование командной строки

### 1. Расширение истории с помощью знака вставки
Предположим, что мы ошиблись в написании команды
```bash
md5sum *.jg | cut -c1-32 | sort | uniq -c | sort -nr
```


Можно исправить тем, что просто ввести старый (неправильный) текст, новый (исправленный) текст и пару знаков вставки `(^)`, например:
```bash
^jq^jpg
```
> Этот метод изменяет только первое вхождение исходного кода `(jg)`. Если ваша исходная команда содержала jg более одного раза, *только первый* экземпляр изменится на `jpg`.

БОЛЕЕЕЕ МОЩНАЯ ЗАМЕНА с помощью расширения истории )) 
Можно использвать синтаксис `sed/ed` (`s/source/target/`,  поставив `duble-bang` в начале:
```bash
!!s:/jg/jpg/
```

Можно также начать с любого расширения истории, например, `!md5sum` вызовет последнюю команду, начинающуюся с `md5sum`, и выполнит ту же замену `jg` на `jpg`:
```bash
$ !md5sum:s/jg/jpg/
```

Этот способ может показаться сложным, но иногда он позволяет быстрее достичь цели, чем другие методы редактирования в командной строке. Запустите `man history` для получения полной информации.

### 2. Редактирование командной строки в стиле Emacs

| Действие                  | Сочетание клавишь           | Комментарий                                          |
| ------------------------- | --------------------------- | ---------------------------------------------------- |
| В начало строки           | `Ctrl + A`                  | Прыжок к началу (к `➜`)                              |
| В конец строки            | `Ctrl + E`                  | Прыжок к концу команды                               |
| На слово назад            | `Alt + B`                   | (Back) Переместить курсор на слово назад             |
| На слово вперед           | `Alt + F`                   | (Forward) Переместить курсор на слово вперед         |
| На символ назад/вперед    | `Ctrl + B` / `Ctrl + F`     | Аналог стрелок влево/вправо                          |
| **Удаление и правка**     |                             |                                                      |
| Удалить слово слева       | `Ctrl + W`                  | Удаляет слово перед курсором (очень часто нужно)     |
| Удалить слово справа      | `Alt + D`                   | Удаляет слово после курсора                          |
| Вырезать до конца строки  | `Ctrl + K`                  | (Kill) Всё от курсора до конца исчезает              |
| Вырезать до начала строки | `Ctrl + U`                  | Полезно, если нужно стереть всю строку сразу         |
| Вставить вырезанное       | `Ctrl + Y`                  | (Yank) Вставляет то, что вырезали через `Ctrl+K/W/U` |
| **История**               |                             |                                                      |
| Поиск по истории          | `Ctrl + R`                  | Начни вводить `md5`, чтобы найти старую команду      |
| Последний аргумент        | `Alt + .`                   | Подставляет последний аргумент прошлой команды       |
| Очистить экран            | `Ctrl + L`                  | Аналог команды `clear`, но быстрее                   |
| **Специальное**           |                             |                                                      |
| Отмена (Undo)             | `Ctrl + _`                  | Отменяет последнее изменение в строке                |
| Редактировать в $EDITOR   | `Ctrl + X` затем `Ctrl + E` | Откроет строку в Nano/Vim для правки                 |
Фича, как сменить  сменить комманду, но оставить аргумент..
Например :
```bash
$ md5sum /path/to/very/long/filename.jpg
```

А теперь хочешь сделать что-то с этим же файлом, просто напиши следующую команду и нажми **`Alt + .`**:
```bash
cat [Alt + .]
cat /path/to/very/long/filename.jpg
```


Три техники упрощают использование Linux:
- Безопасное удаление файлов с помощью !$.
- Инкрементальный поиск через Ctrl-R.
- Редактирование командной строки в стиле Emacs.
