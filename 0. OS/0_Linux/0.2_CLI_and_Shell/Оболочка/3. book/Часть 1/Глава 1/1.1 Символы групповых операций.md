
# Символы групповых операций (Globbing / Pattern Matching) в Bash

> **Globbing** — это механизм подстановки имён файлов и директорий по шаблонам *до* выполнения команды.  
> Обратите внимание: это **не регулярные выражения**, а сопоставление имён путей (pathname expansion).

---

# Основные символы

| Символ                | Название                                         | Действие                                                        |
| --------------------- | ------------------------------------------------ | --------------------------------------------------------------- |
| `*`                   | Звёздочка (asterisk)                             | Любое количество символов (включая ноль), **кроме `/`**         |
| `?`                   | Вопросительный знак                              | Ровно **один** символ (кроме `/`)                               |
| `[...]`               | Класс символов                                   | Любой **один** символ из указанного набора                      |
| `[!...]` или `[^...]` | Инвертированный класс                            | Любой **один** символ, **не входящий** в набор                  |
| `{a,b,c}`             | Расширение по фигурным скобкам (brace expansion) | Подставляет `a`, `b`, `c` (не зависит от существования файлов!) |

---

##  Примеры

### `*`
```bash
ls *.txt        # Все файлы с расширением .txt в текущей папке
ls data_*.log   # Файлы вроде data_2024.log, data_backup.log
ls /home/*      # Все элементы в /home (пользовательские папки и т.д.)
```

### `?`
```bash
ls file?.txt    # file1.txt, fileA.txt — но не file10.txt
ls ???.c        # Имена из ровно 3 символов + .c: foo.c, bar.c
```

### `[...]`
```bash
ls [abc]*.txt   # Файлы, начинающиеся с a, b или c
ls report[0-9].md  # report1.md, …, report9.md (но не report10.md)
ls [[:digit:]]*.log  # Имена, начинающиеся с цифры
ls [[:upper:]]*      # Имена, начинающиеся с заглавной буквы
```

**Полезные классы POSIX** (работают внутри `[...]`):
- `[[:alnum:]]` — буквы и цифры
- `[[:alpha:]]` — буквы
- `[[:digit:]]` — цифры
- `[[:lower:]]`, `[[:upper:]]` — строчные/заглавные буквы
- `[[:space:]]` — пробельные символы
- `[[:punct:]]` — знаки препинания

### `{a,b,c}`
```bash
cp file{,.bak}        # cp file file.bak
mkdir -p logs/{jan,feb,mar}
touch file{1..5}.txt  # file1.txt … file5.txt
echo img_{001..010}.png  # от img_001.png до img_010.png
```

> `{}` — **brace expansion**, выполняется _раньше_ globbing и _не требует_ существования файлов.


## Особенности и тонкости
| Важно знать                                         | Пояснение                                                                                                                                     |
| --------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------- |
| `*` не включает файлы, начинающиеся с `.` (скрытые) | Чтобы их включить: `ls .*` **или** включить опцию `shopt -s dotglob`                                                                          |
| `**` (в bash ≥4.3, zsh)                             | Рекурсивный glob: `**/*.py` — все `.py` во всех подкаталогах (включая текущий)                                                                |
| Отключить globbing                                  | Экранируйте: `\*`, `'*'`, `"*"` — тогда символы интерпретируются буквально                                                                    |
| Если совпадений нет                                 | По умолчанию шаблон остаётся **как есть** (например, `ls *.xyz` → ошибка "No such file") — можно изменить поведение через `shopt -s nullglob` |
| Сравнение с `find`                                  | `find` мощнее (поиск по содержимому, размеру, времени и т.д.), globbing — только по имени и структуре каталогов                               |
## Полезные опции Bash  (`shopt`)
```bash
shopt -s globstar     # включить ** (рекурсивный поиск)
shopt -s nullglob     # если совпадений нет — шаблон исчезает (не передаётся в команду)
shopt -s failglob     # ошибка, если нет совпадений
shopt -s dotglob      # `*` включает скрытые файлы (начинающиеся с `.`)
shopt -s extglob      # включить расширенные шаблоны (см. ниже)
```

### Расширенные шаблоны (`extglob`)
После `shopt -s extglob`:

| Шаблон       | Значение                 |
| ------------ | ------------------------ |
| `?(pattern)` | 0 или 1 совпадение       |
| `*(pattern)` | 0 или более              |
| `+(pattern)` | 1 или более              |
| `@(pattern)` | ровно 1                  |
| `!(pattern)` | всё, **кроме** `pattern` |
Примеры:
```bash
ls !(test*).py     # все .py, кроме начинающихся с test
ls +(ab|cd).txt    # ab.txt, cd.txt, abab.txt — но не xyz.txt
ls ?(v1|v2)-data.csv  # data.csv, v1-data.csv, v2-data.csv
```

## Советы
Всегда проверяйте, что подставляется: используйте `echo` перед опасной командой:
```bash
echo rm *.tmp   # → увидите, что будет удалено
```

- Избегайте `rm *` без уверенности — можно потерять данные.
- Используйте `ls -d` для проверки шаблонов с каталогами:
```bash
ls -d /etc/pam*
```
