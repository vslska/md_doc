### Пример 1. Массовое переименование файлов с сдвигом номеров

**Задача**:  
У вас есть файлы с числовыми суффиксами:  
`item03.txt`, `item04.txt`, ..., `item10.txt`.  
Вы хотите **вставить новый элемент** между `02` и `03`, поэтому все существующие файлы с `03` по `10` должны стать `04`–`11`.

**Решение**:
```bash
paste <(seq -w 10 -1 3 | sed 's/.*/item&.txt/') \
      <(seq -w 11 -1 4 | sed 's/.*/item&.txt/') \
| sed 's/^/mv /' \
| bash
```
#### Как это работает?

1. `seq -w 10 -1 3` → генерирует числа **в обратном порядке**: `10`, `09`, ..., `03`  
    → Обратный порядок **обязателен**, чтобы не перезаписать `item04.txt` до того, как он будет перемещён.
2. `sed 's/.*/item&.txt/'` → превращает `10` в `item10.txt`
3. `paste` соединяет два списка в пары:

```bash
item10.txt item11.txt
item09.txt item10.txt
...
item03.txt item04.txt
```

4.  `sed 's/^/mv /'` → делает из каждой строки команду: `mv item10.txt item11.txt`
5. `| bash` — выполняет все команды

> **Почему именно так?**  
> Это **универсальный шаблон** для любых задач, где нужно сдвинуть нумерацию: логи, бэкапы, сканы, фото и т.д.

> **Важно**:  
> Никогда не делайте такое в прямом порядке (`03 → 04`, потом `04 → 05`) — вы потеряете файл!

#### Альтернатива (если поддерживается):
```bash
# В Bash можно использовать массивы и цикл (без внешних вызовов)
for i in {10..3}; do
    mv "item$(printf "%02d" $i).txt" "item$(printf "%02d" $((i+1))).txt"
done
```

### Пример 2. Поиск непарных файлов (например, `.jpg` без `.txt`)

**Задача**:  
В каталоге лежат файлы вида:

- `photo1.jpg`, `photo1.txt`
- `photo2.jpg` ← **нет описания**
- `report.txt` ← **нет изображения**

Нужно найти **все файлы, у которых нет пары**.

#### Решение 1: через `diff`
```bash
diff <(ls *.jpg | sed 's/\.[^.]*$//') \
     <(ls *.txt | sed 's/\.[^.]*$//') \
| grep '^[<>]' \
| awk '/^</{print $2 ".jpg"} /^>/{print $2 ".txt"}'
```

**Что делает `sed 's/\.[^.]*$//'`?**  
Удаляет **всё после последней точки**, даже если точек несколько:

- `vacation.beach.jpg` → `vacation.beach` ✅
- `cut -d. -f1` дало бы `vacation` ❌

#### Решение 2: через подсчёт (`uniq -c`)
```bash
ls *.{jpg,txt} \
| sed 's/\.[^.]*$//' \
| sort \
| uniq -c \
| awk '$1 == 1 {print $2 "*"}' \
| xargs -I{} ls {}
```

**Логика**:
- Если имя встречается **дважды** → есть и `.jpg`, и `.txt`
- Если **один раз** → файл без пары
- Добавляем `*` → получаем шаблон (`photo2*`)
- `xargs ls {}` → показывает реальные файлы

> **Плюсы второго способа**:> 
> - Не зависит от порядка файлов
> - Легко адаптируется под другие расширения (`*.log` + `*.yaml`, например)

> **Минус обоих**:  
> Использование `ls` небезопасно в скриптах (ломается при пробелах или отсутствии файлов).  
> Для production лучше использовать `find` или glob-массивы.

### Универсальные принципы

1.  **Генерируйте данные, а не пишите вручную**  
    → `seq`, `{1..10}`, `find` — ваши друзья.
2. **Преобразуйте по шагам**  
    Каждая команда в конвейере — фильтр:  
    `данные → форматирование → объединение → выполнение`
3.  **Тестируйте перед запуском**  
    Замените `| bash` на `| less` или добавьте `echo`:
```bash
... | sed 's/^/echo mv /' | head
```

4. **Обрабатывайте сложные имена**  
    Всегда предполагайте, что в имени файла могут быть:
    - пробелы,
    - точки,
    - спецсимволы (`$`, `(`, `[`).
5. **Избегайте `ls` в скриптах**  
    Вместо `ls *.jpg` используйте:
```bash
for f in *.jpg; do [[ -e "$f" ]] && echo "$f"; done
```

### Когда использовать такие однострочники?

✅ **Хорошо**:

- Разовые задачи (переименовать 50 файлов)
- Анализ данных в терминале
- Быстрая автоматизация без написания скрипта

❌ **Плохо**:

- Production-логика
- Сложная бизнес-логика
- Повторяющиеся задачи → лучше написать скрипт
