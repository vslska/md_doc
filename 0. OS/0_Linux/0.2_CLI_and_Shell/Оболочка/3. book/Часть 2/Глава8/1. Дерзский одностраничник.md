### Создание дерзких однострочников

**Дерзкий однострочник** (brash one-liners) — это компактная, часто многоуровневая команда, которая решает нетривиальную задачу за один запуск. Она выглядит как «магия», но на самом деле — результат чёткого мышления и глубокого знания инструментов.

> Это не ради хвастовства. Это тренировка:  
> -   умения разбивать задачу на шаги,  
> -  понимания потоков данных,  
> - владения конвейерами и подстановками.

#### Пример
``` bash
paste <(echo {1..10}.jpg | sed 's/ /\n/g') \
      <(echo {0..9}.jpg  | sed 's/ /\n/g') \
| sed 's/^/mv /' \
| bash
```

> **Цель**: переименовать `1.jpg → 0.jpg`, `2.jpg → 1.jpg`, ..., `10.jpg → 9.jpg`.

**По шагам**:
1. `{1..10}.jpg` → `1.jpg 2.jpg ... 10.jpg` (одна строка)
2. `sed 's/ /\n/g'` → превращает в столбец
3. `paste` соединяет два столбца:
```bash
1.jpg  0.jpg
2.jpg  1.jpg
...
```
4. `sed 's/^/mv /'` → делает из каждой строки команду:  
    `mv 1.jpg 0.jpg`
5. `| bash` — выполняет сгенерированные команды.

> Всё в одной строке. Никаких циклов. Никаких временных файлов.

---
### Методология создания однострочников

#### 1. **Начну с исходных данных**

Что у меня есть?

- Список файлов? → `ls`, `find`
- Диапазон чисел? → `{1..100}`
- Дата? → `date`
- Содержимое файла? → `cat`, `grep`

**Пример**: найти 17-ю букву алфавита:
```bash
echo {A..Z} | tr ' ' '\n' | sed -n '17p'
# или
echo {A..Z} | awk '{print $17}'
```

#### 2. **Преобразуй данные по шагам**

Используй конвейер:

- `tr`, `sed` — форматирование
- `awk`, `cut` — извлечение
- `sort`, `uniq`, `head`/`tail` — фильтрация
- `xargs`, `paste` — перекомпоновка

#### 3. **Проверяй каждый этап**

Не пишу всё сразу. Собираю по кирпичикам:
```
# Шаг 1
echo {1..5}

# Шаг 2
echo {1..5} | tr ' ' '\n'

# Шаг 3
paste <(echo {1..5}) <(echo {6..10})

# Шаг 4 — готово!
paste <(echo {1..5}) <(echo {6..10}) | sed 's/^/echo pair: /'
```

#### 4. **Безопасное выполнение**
Перед `| bash` всегда проверю:
```bash
# ❌ Опасно
... | bash

# ✅ Безопасно
... | less        # смотрим
... | head -n3    # проверяем начало
... | sed 's/^/echo /' | bash  # dry-run
```

### Инструменты для отладки
| Приём                   | Зачем                                              |
| ----------------------- | -------------------------------------------------- |
| `echo перед командой`   | Dry-run для `rm`, `mv`, `cp`                       |
| `tee debug.txt`         | Посмотреть промежуточный вывод в длинном пайплайне |
| `set -x`                | Включить трассировку в скрипте                     |
| История (`↑`, `Ctrl-R`) | Быстро повторять и править команды                 |

```bash
find . -name "*.log" \
| tee /tmp/logs_found.txt \
| xargs -r rm
# Теперь можно проверить: cat /tmp/logs_found.txt
```

### Фишки и альтернативные подходы

#### Гибкость — ключ

Задача: показать все `.jpg` файлы.

Можно **многими способами**:
```bash
ls *.jpg
ls | grep '\.jpg$'
find . -maxdepth 1 -name "*.jpg"
compgen -G "*.jpg"          # только имена, без ls
printf '%s\n' *.jpg         # безопаснее, если файлов много
```

Выбирай **под контекст**:

- `*.jpg` ломается при большом числе файлов → используй `find`
- Нужны только имена без метаданных → `printf` или `compgen`

#### Генерация команд через `awk`

Хочу создать 10 команд `touch file1.txt ... file10.txt`?

```bash
seq 10 | awk '{print "touch file" $0 ".txt"}' | bash
```

#### Использование `xargs -I` для сложных шаблонов
```bash
find . -name "*.bak" -print0 | xargs -0 -I {} sh -c 'echo "Удаляю: {}"; rm "{}"'
```

### Когда НЕ стоит писать однострочник

- Логика сложная (ветвления, циклы с условиями) → пиши скрипт.
- Команда будет использоваться часто → вынеси в функцию или скрипт.
- Безопасность критична → лучше явный, читаемый код.

> **Правило**:  
   Если мне самому через неделю будет непонятно, что делает команда — она слишком «дерзкая».
### Итог

Дерзкие однострочники — это:

- **Искусство композиции** стандартных утилит,
- **Тренировка системного мышления**,
- **Инструмент быстрого прототипирования**.
