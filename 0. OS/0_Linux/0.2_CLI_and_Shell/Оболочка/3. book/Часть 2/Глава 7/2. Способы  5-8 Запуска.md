### Команда как строка

Всё, что вы вводите в оболочку, — это **текстовая строка**, которую интерпретатор разбирает и выполняет. Но иногда бывает полезно **собрать команду динамически** — как строку — и затем передать её на выполнение. Это открывает мощные возможности для автоматизации, но требует особой осторожности: **выполняемая строка — это исполняемый код**, и ошибка может привести к непредсказуемым последствиям (вплоть до удаления важных данных).

>**Правило безопасности**:  
   Никогда не выполняйте сгенерированную строку вслепую. Всегда сначала **выведите её на экран**, убедитесь, что она корректна, и только потом запускайте.

Рассмотрим четыре основных способа выполнения команды, представленной в виде строки.

---
#### Cпособ #5. Передача команды в `bash -c`

Команда `bash -c "строка"` запускает новую **дочернюю оболочку**, которая выполняет указанную строку как команду, а затем завершается.

Это особенно полезно в двух случаях:

1. **Изоляция изменений**  
Любые изменения (каталог, переменные, функции) остаются внутри дочерней оболочки:
```bash
$ pwd
/home/user
$ bash -c "cd /tmp && touch temp_file"
$ pwd
/home/user  # текущий каталог не изменился
```

2. **Правильное использование `sudo` с перенаправлением**  
Проблема:
```bash
sudo echo "log" > /var/log/app.log  # ❌ ОШИБКА!
```
Почему? Потому что перенаправление (`>`) обрабатывается **локальной оболочкой от имени обычного пользователя**, ещё до запуска `sudo`. Поэтому вы получаете `Permission denied`.

Решение — передать **всю команду целиком** в оболочку, запущенную под `sudo`:
```bash
sudo bash -c 'echo "log" > /var/log/app.log'  # ✅ РАБОТАЕТ
```

>1. Обратите внимание: внутри `bash -c` используется **одинарная кавычка**, чтобы предотвратить преждевременное раскрытие переменных или спецсимволов в текущей оболочке.

#### Способ #6. Передача команды через стандартный ввод (`| bash`)

Оболочка может читать команды не только с клавиатуры, но и из **стандартного ввода**. Это позволяет генерировать команды программно и выполнять их пакетно.

Пример: автоматическая сортировка файлов по первой букве имени:
```bash
# Создаём подкаталоги a–z
mkdir {a..z}

# Генерируем команды mv
ls ??* | sed 's/^\(.\)\(.*\)$/mv "\1\2" \1/'

# Проверяем результат (обязательно!)
ls ??* | sed 's/^\(.\)\(.*\)$/mv "\1\2" \1/' | less

# Выполняем
ls ??* | sed 's/^\(.\)\(.*\)$/mv "\1\2" \1/' | bash
```

>Этот подход следует шаблону:  
**1. Сгенерировать → 2. Проверить → 3. Выполнить**

Он особенно мощен при работе с большими наборами данных, где ручной ввод невозможен.

#### Способ #7. Удалённое выполнение через `ssh`

Команда `ssh` позволяет не только войти на удалённый хост, но и **выполнить одну команду** без интерактивной сессии:
```bash
ssh user@server "df -h"
```
Это критически важно для скриптов мониторинга, развёртывания и резервного копирования.
##### Важные нюансы:
- **Перенаправление выполняется на том хосте, где интерпретируется команда**:
```bash
ssh server "ls > remote.txt"   # файл создаётся на сервере
ssh server ls > local.txt      # файл создаётся локально
```

- **Подавление лишнего вывода** (например, banner'ов):
```bash
  echo "uptime" | ssh -T server bash
```
- Флаг `-T` отключает выделение псевдотерминала, а явный вызов `bash` часто подавляет приветственные сообщения.
- **Безопасность**: используйте SSH-ключи, чтобы избежать запросов пароля в скриптах.

#### Способ #8. Массовое выполнение через `xargs`

Команда `xargs` принимает список строк (обычно из stdin) и **подставляет их как аргументы** в заданную команду.
##### Базовый пример:
```bash
ls *.txt | xargs wc -l
```

>→ эквивалентно `wc -l file1.txt file2.txt ...`

Но у этого подхода есть **фатальный недостаток**: он ломается при именах файлов с пробелами, переводами строк и другими спецсимволами.
##### Безопасное использование: `find -print0 | xargs -0`
```bash
find . -name "*.log" -print0 | xargs -0 gzip
```

- `-print0` в `find` разделяет пути **нулевым байтом** (`\0`)
- `-0` в `xargs` говорит: «используй `\0` как разделитель»

Это **единственный надёжный способ** обрабатывать произвольные имена файлов.

##### Полезные опции `xargs`:

- `-n N` — выполнять команду с **не более чем N аргументами** за раз
```bash
seq 1 6 | xargs -n 2 echo  # → echo 1 2; echo 3 4; echo 5 6
```

- `-I {}` — вставить каждый аргумент в **произвольное место** команды
```bash
find . -name "*.conf" -print0 | xargs -0 -I {} cp {} {}.bak
```

- Обход ограничения «Argument list too long»:  
При попытке `rm *.txt` в каталоге с миллионом файлов вы получите ошибку.  
Решение:
```bash
find . -maxdepth 1 -name "*.txt" -print0 | xargs -0 rm
```

> Альтернатива: `find -exec command {} +` — делает то же самое, что и `xargs`, но встроен в `find`. Однако `xargs` даёт больше контроля (например, через `-I`)
