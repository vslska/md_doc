### Способ #10. Явные подоболочки `(...)`

Круглые скобки `(команда)` запускают **подоболочку** — временную копию текущей оболочки, которая:

- наследует **всё**: переменные, псевдонимы, функции, текущий каталог;
- изолирована: любые изменения **не влияют на родительскую оболочку**;
- завершается сразу после выполнения.

#### Зачем это нужно?

##### 1. **Временная смена каталога без `cd ..`**
```bash
$ pwd
/home/user
$ (cd /tmp && ls)
file1.txt  file2.log
$ pwd
/home/user  # остались на месте
```

##### 2. **Извлечение архива в другой каталог**
```bash
tar xzf archive.tar.gz -C /target/dir
# или, если `-C` недоступен:
cat archive.tar.gz | (mkdir -p /target && cd /target && tar xzf -)
```

##### 3. **Копирование через `tar` + подоболочка**

```bash
# Локально
tar czf - dir/ | (cd /dest && tar xzf -)

# Удалённо через SSH
tar czf - dir/ | ssh user@host '(cd /dest && tar xzf -)'
```

> → Это **надёжнее**, чем `scp -r`, особенно при сложных правах или символических ссылках.

##### 4. **Изоляция переменных**
```bash
$ VAR=original
$ (VAR=modified; echo "Inside: $VAR")
Inside: modified
$ echo "Outside: $VAR"
Outside: original
```

>Подоболочки — **не процессы**, а лёгкие клонированные контексты.  
   Проверить можно через `$BASH_SUBSHELL`:
```bash
echo $BASH_SUBSHELL        # 0
(echo $BASH_SUBSHELL)      # 1
```

### Способ #11. Замена процесса: `exec`

Команда `exec` **заменяет текущую оболочку** новым процессом. После завершения — **оболочка не возвращается**.

#### Основные применения

##### 1. **Оптимизация скриптов**

Если последняя команда в скрипте — внешняя программа, используйте `exec`, чтобы избежать лишнего процесса:
```bash
#!/bin/bash
# ... подготовка ...
exec python3 app.py  # заменяет bash на python
```

> Экономия памяти и PID при массовом запуске (например, в CI/CD или контейнерах).

##### 2. **Перенаправление всего вывода скрипта**
```bash
#!/bin/bash
exec > /var/log/myscript.log 2>&1
echo "Это попадёт в лог"
ls /nonexistent  # ошибка тоже в лог
```

> Не нужно писать `>> logfile` после каждой команды.

##### 3. **Перенаправление в файловые дескрипторы (продвинуто)**
```bash
exec 3> debug.log
echo "Отладка" >&3
exec 3>&-  # закрыть дескриптор
```
#### Осторожно!
```bash
exec ls
```

>  — после этого **терминал закроется**, потому что оболочка заменена на `ls`, а когда `ls` завершится — родительского процесса нет.

> Используйте `exec` только в скриптах или временных сессиях. Никогда — в интерактивной оболочке, если не хотите вылететь из терминала.

### Быстрый справочник: когда что использовать?
| Задача                                           | Рещение                                 |
| ------------------------------------------------ | --------------------------------------- |
| Временно сменить каталог                         | `(cd dir && command)`                   |
| Выполнить команду с изоляцией                    | `(VAR=value; cmd)`                      |
| Перенаправить весь вывод скрипта                 | `exec > log.txt 2>&1`                   |
| Запустить последнюю команду без лишнего процесса | `exec program`                          |
| Копировать дерево каталогов надёжно              | `tar cf - src \| (cd dest && tar xf -)` |

---
### Итог

- **Подоболочки `(...)`** — ваш инструмент для **временной изоляции** без побочных эффектов.
- **`exec`** — мощный, но опасный способ **заменить оболочку** или **перенаправить всё** в скрипте.
