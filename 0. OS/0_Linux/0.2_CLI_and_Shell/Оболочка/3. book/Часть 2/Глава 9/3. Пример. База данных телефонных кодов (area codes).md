### Пример 3. База данных телефонных кодов (area codes)

 [Areacodes](https://resources.oreilly.com/examples/0636920601098/-/blob/master/ch09/build_area_code_database/areacodes.txt?ref_type=heads)

**Задача**:  
Хранить и быстро искать информацию о телефонных кодах США: код → штат → города.

**Решение**:  
Использовать **структурированный текстовый файл** с табуляцией в качестве разделителя:
```
201	NJ	"Hackensack, Jersey City"
202	DC	Washington
...
```

>  Почему **табуляция**, а не запятая?  
> Потому что в названиях городов есть запятые (`Boston, Cambridge`), а табуляция почти никогда не встречается в таких данных.

---

#### Основные операции

##### 1. Поиск по штату
```bash
grep -w "CA" areacodes.txt
```
> `-w` гарантирует, что найдётся только целое слово `CA`, а не `CAL` или `SCA`.

##### 2. Поиск по коду
```bash
grep -w "202" areacodes.txt
```

##### 3. Подсчёт количества кодов на штат
```bash
cut -f2 areacodes.txt | sort | uniq -c | sort -nr | head -5
```
> Выведет топ-5 штатов по числу кодов.

##### 4. Экспорт в CSV (для Excel, LibreOffice)
```bash
awk -F'\t' '{printf "%s,%s,\"%s\"\n", $1, $2, $3}' areacodes.txt > areacodes.csv
```
> Города в кавычках — запятые внутри не ломают структуру.

##### 5. Собрать все коды по штату
```bash
awk '{codes[$2] = codes[$2] " " $1} END {for (st in codes) print st ":" codes[st]}' areacodes.txt | sort
```
> Результат: `CA: 209 213 310 323 ...`

---

###  Улучшения и фишки

#### 1. **Безопасный формат файла**
Храните данные в **TSV** (tab-separated values), а не в «выровненных колонках» — это надёжнее для обработки.

Пример правильного формата (`areacodes.tsv`):
```tsv
201	NJ	Hackensack, Jersey City
202	DC	Washington
203	CT	New Haven, Stamford
```

#### 2. **Универсальный скрипт поиска**
```bash
#!/bin/bash
# ~/bin/areacode
if [[ -n "$1" ]]; then
    grep -iE "(^|[[:space:]])$1([[:space:]]|$)" areacodes.tsv
else
    echo "Usage: areacode <code|state|city>"
fi
```
> Ищет **точное совпадение** любого поля, без ложных срабатываний.

#### 3. **Поиск по нескольким критериям**
Найти все коды в Калифорнии, связанные с Лос-Анджелесом:
```bash
awk -F'\t' '$2=="CA" && $3 ~ /Los Angeles/' areacodes.tsv
```

#### 4. **Проверка дубликатов**
```bash
sort areacodes.tsv | uniq -d
```
> Найдёт повторяющиеся строки.

#### 5. **Генерация справочника для терминала**
Создайте функцию в `.bashrc`:
```bash
ac() {
    awk -F'\t' -v q="$1" '$1==q || $2==q || $3~q' ~/data/areacodes.tsv
}
```
> Теперь можно: `ac 213` или `ac TX`.

---

###  Ограничения и ловушки

- **Регулярные выражения в `grep`** могут давать ложные совпадения → используйте `-w` или `awk` с точным сравнением.
- **Регистр**: `ca` ≠ `CA` → добавьте `-i` или нормализуйте данные.
- **Многострочные значения** (редко, но бывает) → TSV не поддерживает, лучше использовать JSON.

---

### Альтернативы

- Для сложных запросов — загрузите данные в SQLite:
  ```sql
  CREATE TABLE codes (area INT, state TEXT, cities TEXT);
  .import --csv areacodes.csv codes
  SELECT * FROM codes WHERE state = 'CA';
  ```
- Для веб-интерфейса — экспортируйте в JSON и используйте простой Python-скрипт.

### Итог

Этот пример показывает, как **простой текстовый файл** превращается в:
- базу данных,
- инструмент поиска,
- источник для отчётов.

Вы не используете СУБД, не пишете веб-приложение — просто применяете:
- `grep` — фильтрация,
- `awk` — преобразование,
- `sort`/`uniq` — агрегация.

> ✅ **Правило**:  
> Если данные можно представить в виде строк — вы уже на 80% решили задачу.  
> Остальное — дело техники.
