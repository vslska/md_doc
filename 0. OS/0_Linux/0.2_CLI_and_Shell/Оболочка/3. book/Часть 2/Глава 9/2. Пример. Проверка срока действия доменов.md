### Пример 2. Проверка срока действия доменов

**Задача**: отслеживать даты окончания регистрации нескольких доменов.

**Решение**:  
1. Хранить список доменов в текстовом файле.  
2. Извлекать дату через `whois`.  
3. Обрабатывать и анализировать результаты как структурированные данные.

---

#### Пошаговая реализация

1. **Создать файл `domains.txt`**  
   ```text
   example.com
   oreilly.com
   efficientlinux.com
   ```

2. **Извлечь дату окончания для одного домена**  
   ```bash
   whois example.com | grep 'Registry Expiry Date:' | awk '{print $4}'
   # → 2022-08-13T04:00:00Z
   ```

3. **Преобразовать в читаемый формат**  
   ```bash
   date --date="$(whois example.com | awk '/Registry Expiry Date:/ {print $4}')" '+%Y-%m-%d'
   # → 2022-08-13
   ```

4. **Скрипт `check-expiry`**  
   ```bash
   #!/bin/bash
   domain="$1"
   exp=$(whois "$domain" 2>/dev/null | awk '/Registry Expiry Date:/ {print $4}')
   if [[ -n "$exp" ]]; then
       date --date="$exp" '+%Y-%m-%d' 2>/dev/null || echo "invalid"
   else
       echo "unknown"
   fi
   ```

5. **Обработать все домены**  
   ```bash
   while read -r domain; do
       [[ -n "$domain" ]] && echo "$(./check-expiry "$domain")	$domain"
       sleep 5  # уважение к whois-серверам
   done < domains.txt > expiry.txt
   ```

> ⚠️ **Важно**:  
> Многие регистраторы **ограничивают частоту запросов**. Без `sleep` вы можете получить бан или ошибку.

---

### 💡 Улучшения и фишки

#### 1. **Безопасность и обработка ошибок**
- Добавьте `2>/dev/null`, чтобы подавить ошибки `whois` (например, при недоступности домена).
- Проверяйте, что дата извлечена: `if [[ -n "$exp" ]]`.

#### 2. **Сортировка по дате окончания**
```bash
sort -k1,1 expiry.txt
```
→ Самые «горячие» домены — сверху.

#### 3. **Найти просроченные или близкие к окончанию**
```bash
# Все, что истекает сегодня или раньше
awk -v today="$(date +%Y-%m-%d)" '$1 <= today' expiry.txt

# Истекает в ближайшие 14 дней
awk -v today="$(date -d '+14 days' +%Y-%m-%d)" '$1 <= today' expiry.txt
```

#### 4. **Автоматизация через cron**
Добавьте в `crontab`:
```bash
0 9 * * 1 /path/to/check-expiry-all.sh | mail -s "Domain Expiry Report" admin@example.com
```
→ Отчёт каждое утро понедельника.

#### 5. **Альтернатива: `jq` + API (для массовой проверки)**
Если у вас десятки доменов — лучше использовать API регистратора (например, Cloudflare, GoDaddy) вместо `whois`. Но для 5–10 доменов `whois` — идеален.

---

### ⚠️ Ограничения `whois`

- Формат вывода **зависит от регистратора**. Некоторые не показывают `Registry Expiry Date`.
- Для `.ru`, `.de`, `.uk` и других TLD — может потребоваться другой шаблон (`grep 'Expiry Date'`, `Expiration Date:` и т.д.).
- Некоторые домены скрывают данные (privacy protection).

> ✅ **Совет**:  
> Протестируйте `whois your-domain.tld` вручную, чтобы уточнить формат перед автоматизацией.

---

### Итог

Этот пример демонстрирует мощь подхода:
> **Текстовый файл → стандартные утилиты → автоматизированный отчёт**

Вы не пишете сложный код — просто комбинируете:
- `whois` — источник данных,
- `awk`/`grep` — извлечение,
- `date` — нормализация,
- `sort`/`awk` — анализ.

И получаете **готовую систему мониторинга** за 10 строк скрипта.

> 🔔 Главное правило:  
> **Уважайте чужие серверы. Всегда добавляйте `sleep` при массовых запросах.**
