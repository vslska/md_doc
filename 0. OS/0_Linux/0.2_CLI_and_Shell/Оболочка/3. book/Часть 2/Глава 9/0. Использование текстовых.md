### Использование текстовых файлов: данные как код

В Linux **всё — текст**:
- конфигурации (`/etc/*.conf`),
- системные базы (`/etc/passwd`, `/etc/group`),
- логи,
- исходный код,
- письма, заметки, списки задач.

Даже если сегодня всё «в облаке» — **локальная обработка текста** остаётся мощнейшим инструментом для анализа, автоматизации и отладки.

>  Главная идея:  
> **Структурированный текст + стандартные утилиты = готовое решение без программирования.**

---

#### Пример: массовая рассылка пользователям

**Задача**: отправить приветствие всем обычным пользователям (UID ≥ 1000).

**Решение по шагам**:

1. **Извлечь имена из `/etc/passwd`**  
   ```bash
   awk -F: '$3 >= 1000 {print $1}' /etc/passwd
   ```
   > `-F:` задаёт разделитель полей (двоеточие)

2. **Сформировать приветствие**  
   ```bash
   xargs -I@ echo "Hi there, @!"
   ```

3. **Сгенерировать команды отправки**  
   ```bash
   xargs -I@ echo 'echo "Hi there, @!" | mail -s greetings @'
   ```

4. **Выполнить**  
   ```bash
   ... | bash
   ```

>  **Предупреждение**:  
> Такой подход **не масштабируется** и **небезопасен** при наличии спецсимволов в именах.  
> В production используйте циклы или скрипты:
> ```bash
> while IFS=: read -r user _ uid _; do
>   [[ $uid -ge 1000 ]] && echo "Hi $user" | mail -s "greetings" "$user"
> done < /etc/passwd
> ```

###  Методология работы с текстовыми данными

1. **Проанализируйте задачу**  
   Что нужно получить? Какие данные есть?

2. **Сохраните данные в структурированном виде**  
   Лучшие форматы:
   - CSV (разделитель — запятая/табуляция)
   - TSV (табуляция — надёжнее запятой)
   - одно значение на строку
   - фиксированная структура (как в `/etc/passwd`)

3. **Обработайте стандартными утилитами**  
   - `cut` — извлечение полей (просто, но жёстко)
   - `awk` — гибкая обработка (лучше для условий)
   - `grep` — фильтрация
   - `sort`, `uniq` — агрегация
   - `join` — объединение двух файлов по ключу

4. **Автоматизируйте (опционально)**  
   Вынесите в функцию, псевдоним или скрипт.

###  Практические примеры

#### 1. Найти всех пользователей с оболочкой `/bin/bash`
```bash
awk -F: '$7 == "/bin/bash" {print $1}' /etc/passwd
```

#### 2. Подсчитать домашние каталоги по дисковому использованию
```bash
awk -F: '$3 >= 1000 {print $6}' /etc/passwd | xargs -I{} du -sh {}
```

#### 3. Создать CSV из `/etc/passwd`
```bash
awk -F: 'BEGIN{OFS=","} {print $1,$3,$6}' /etc/passwd > users.csv
```

#### 4. Найти «висячие» домашние каталоги (без пользователя)
```bash
# Получить список UID из passwd
uids=$(awk -F: '$3>=1000{print $3}' /etc/passwd | sort -u)

# Сравнить с владельцами каталогов в /home
for dir in /home/*; do
  owner_uid=$(stat -c %u "$dir")
  if ! grep -q "^$owner_uid$" <<< "$uids"; then
    echo "Orphaned: $dir"
  fi
done
```

###  Ограничения и ловушки

| Проблема | Решение |
|--------|--------|
| Имена с пробелами/спецсимволами | Используйте `while read` вместо `for` |
| Большие файлы | Избегайте `cat file \| ...`, читайте напрямую: `awk '...' file` |
| Неоднозначные разделители | Предпочитайте TSV или JSON (с `jq`) |
| Отсутствие структуры | Сначала нормализуйте данные (через `sed`, `awk`) |

---

### Альтернативы для сложных данных

- **JSON** → `jq`  
  ```bash
  curl -s https://api.example.com/users | jq -r '.[].email'
  ```

- **YAML** → `yq` (не входит в базовую систему)

- **Базы данных** → `sqlite3` (лёгковесная SQL-база, работает с файлами)

Но помните:  
> **Если данные можно представить как строки — вы уже выиграли.**

### Итог

Текстовые файлы — не пережиток прошлого, а **основа Unix-философии**:
> «Пиши программы, которые делают одно и делают это хорошо. Пиши программы, чтобы они работали вместе. Пиши программы, которые обрабатывают текстовые потоки, потому что это универсальный интерфейс.»

Освоив обработку текста через `awk`, `cut`, `grep`, `sort` — вы получаете **универсальный инструмент для анализа любых структурированных данных**, без зависимости от языков программирования или GUI.
