### Пример 4. Создание простого менеджера паролей (`pman`)

> ⚠️ **Важно**: это **учебный пример**. Не используйте его для хранения реальных паролей без понимания рисков.

---

#### Шаг 1. Создание файла `vault`

Файл `vault` — это **простой текстовый файл** в формате TSV (tab-separated values).  
Каждая запись занимает **одну строку** и состоит из **трёх полей**, разделённых **табуляцией**:

```
<username>	<password>	<tag>	<notes>
```

> 💡 На самом деле в исходном примере используется **3 поля**, но часто удобнее сделать **4**:  
> - логин  
> - пароль  
> - короткий ключ (например, `google`, `bank`)  
> - примечания

Но в книге — именно **3 поля**, где третье поле — и есть **ключ + примечания**.

##### Пример содержимого:
```text
sally	fake1	google google.com account
ssmith	fake2	dropbox dropbox.com account for work
s999	fake3	bank Bank of America account, bankofamerica.com
```

> 🔍 Обрати внимание: между `google` и `google.com account` — **пробел**, а не табуляция.  
> Это означает, что **весь остаток строки — примечание**, а **первое слово в третьем поле — ключ**.

---

#### Шаг 2. Безопасное создание и настройка прав

```bash
# 1. Создать пустой файл
touch ~/etc/vault

# 2. Установить строгие права: только владелец может читать/писать
chmod 600 ~/etc/vault

# 3. Отредактировать (лучше через vim/emacs, не через echo!)
vim ~/etc/vault
```

> ✅ Почему `600`?  
> - `6` = `rw-` для владельца  
> - `0` = `---` для группы и остальных  
> → Никто кроме вас не сможет прочитать файл.

> ⚠️ **Никогда не используйте `echo "..." > vault`** — это легко оставить в истории терминала!

---

#### Шаг 3. Базовый скрипт `pman` (версия 1)

Сохраните в `~/bin/pman`:
```bash
#!/bin/bash
grep "$1" "$HOME/etc/vault"
```

Сделайте исполняемым:
```bash
chmod 700 ~/bin/pman
```

> ✅ `700` — только владелец может запускать.

Теперь можно искать:
```bash
pman google
# → найдёт любую строку, содержащую "google"
```

#### Шаг 4. Улучшение: проверка аргументов и ошибок (версия 2)

```bash
#!/bin/bash
PROGRAM=$(basename "$0")
DATABASE="$HOME/etc/vault"

if [ $# -ne 1 ]; then
    >&2 echo "$PROGRAM: look up passwords by string"
    >&2 echo "Usage: $PROGRAM <search-term>"
    exit 1
fi

searchstring="$1"
grep "$searchstring" "$DATABASE"
if [ $? -ne 0 ]; then
    >&2 echo "$PROGRAM: no matches for '$searchstring'"
    exit 1
fi
```

→ Теперь скрипт не падает молча и даёт понятные сообщения.

---

#### Шаг 5. Улучшение: приоритет поиску по **ключу** (версия 3)

Измените формат `vault`, чтобы **третье поле начиналось с уникального ключа**:

```text
sally	fake1	google	google.com account
ssmith	fake2	dropbox	dropbox.com for work
```

Теперь скрипт сначала ищет **точное совпадение по ключу**:
```bash
match=$(awk -v key="$searchstring" '$3 == key' "$DATABASE")
```

Если не найдено — ищет по всему файлу.

---

#### Шаг 6. Шифрование через GPG

1. **Создайте GPG-ключ** (если нет):
   ```bash
   gpg --quick-generate-key your@email.com
   ```

2. **Зашифруйте файл**:
   ```bash
   cd ~/etc
   gpg -e -r your@email.com vault
   # → создаёт vault.gpg
   ```

3. **Проверьте расшифровку**:
   ```bash
   gpg -d vault.gpg
   ```

4. **Удалите исходный файл** (после проверки!):
   ```bash
   shred -u vault  # безопасное удаление
   ```

#### Шаг 7. Финальная версия `pman` (версия 4)

```bash
#!/bin/bash
PROGRAM=$(basename "$0")
DATABASE="$HOME/etc/vault.gpg"

if [ $# -ne 1 ]; then
    >&2 echo "$PROGRAM: look up passwords"
    >&2 echo "Usage: $PROGRAM <search-term>"
    exit 1
fi

searchstring="$1"

# Расшифровываем и фильтруем комментарии (#)
decrypted=$(gpg -d -q "$DATABASE" 2>/dev/null | grep -v '^#')

# Ищем по ключу (3-е поле)
match=$(echo "$decrypted" | awk -v key="$searchstring" '$3 == key')

# Если не найдено — ищем по всему тексту
if [ -z "$match" ]; then
    match=$(echo "$decrypted" | awk "/$searchstring/")
fi

if [ -z "$match" ]; then
    >&2 echo "$PROGRAM: no matches for '$searchstring'"
    exit 1
fi

echo "$match"
```

#### Шаг 8. Дополнительные улучшения

##### Поддержка комментариев
Добавьте в `vault.gpg` строки вида:
```text
# Личные аккаунты
sally	fake1	google	google.com
```

Скрипт автоматически их игнорирует благодаря `grep -v '^#'`.

##### Безопасный вывод (рекомендация)
Вместо `echo "$password"` — копируйте в буфер:
```bash
echo "$password" | xclip -selection clipboard
echo "Пароль скопирован. Очистка через 15 сек..."
(sleep 15; echo -n | xclip -selection clipboard) &
```

##### Прямое редактирование
Настройте `vim` или `emacs` для работы с `.gpg`-файлами (см. книгу), или используйте:
```bash
alias pwedit='vim ~/.etc/vault.gpg'
```

Не забудьте:
```bash
export GPG_TTY=$(tty)
```
— иначе GPG не сможет запросить пароль в фоне.

### Итог: что мы получили?

- **Структурированное хранилище** в виде текста.
- **Быстрый поиск** по ключу или любому тексту.
- **Шифрование** через промышленный стандарт (GPG).
- **Простота** — всё на базе `awk`, `grep`, `gpg`.

> 🧠 Главный урок:  
> **Даже чувствительные данные можно хранить в тексте — если правильно организовать структуру и защиту.**

Но помните: для реального использования лучше выбрать **специализированный менеджер** (KeePassXC, Bitwarden). Этот пример — для понимания принципов.
