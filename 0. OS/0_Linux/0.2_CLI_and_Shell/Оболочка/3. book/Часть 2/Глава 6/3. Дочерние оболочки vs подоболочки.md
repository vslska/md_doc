### Дочерние оболочки vs подоболочки

Когда вы запускаете новую оболочку командой `bash`, создаётся **дочерний процесс**. Он получает копию **переменных окружения** родителя (только те, что были экспортированы через `export`), но **не наследует** локальные переменные, псевдонимы (`alias`) или функции.

Пример:
```bash
$ alias l='ls -CF'
$ l                    # работает
$ bash --norc          # запускаем дочернюю оболочку без .bashrc
$ l                    # ошибка: команда не найдена
$ echo $HOME           # работает — HOME экспортирована
/home/user
$ exit
```

>Именно поэтому **псевдонимы не работают внутри скриптов**: скрипты выполняются в дочерних оболочках, которые не видят псевдонимов родительской сессии.

#### Подоболочка — полная копия состояния

В отличие от дочерней оболочки, **подоболочка** — это не отдельный процесс, а временное ответвление текущей оболочки. Она наследует **всё**:

- локальные и экспортные переменные,
- псевдонимы,
- функции,
- текущий каталог,
- опции оболочки.

Создаётся подоболочка с помощью круглых скобок:
```bash
$ alias ll='ls -l'
$ (ll)                 # работает — псевдоним доступен
-rw-r--r-- 1 user user 0 Jan 25 10:00 file.txt
```

Подоболочка особенно полезна, когда нужно **временно изменить окружение**, не затрагивая родительскую сессию:
```bash
$ pwd
/home/user
$ (cd /tmp && pwd)     # меняем каталог только внутри подоболочки
/tmp
$ pwd                  # остались на месте
/home/user
```

#### Как отличить?

Оболочка предоставляет специальную переменную — `BASH_SUBSHELL`:

- В обычной оболочке и в дочерней оболочке её значение — `0`.
- В подоболочке — `1` (или больше при вложенных подоболочках).
```bash
$ echo $BASH_SUBSHELL
0
$ bash
$ echo $BASH_SUBSHELL    # всё ещё 0 — это дочерняя оболочка
0
$ exit
$ (echo $BASH_SUBSHELL)  # теперь 1 — подоболочка
1
```

>**Примечание**: подоболочки — это особенность Bash и Zsh. В POSIX-совместимом `/bin/sh` поведение может отличаться или быть недоступным.

#### Практический вывод

- Используйте **подоболочки** (`(...)`) для временных операций с изоляцией (например, смена каталога, установка временных переменных).
- Помните: **скрипты и дочерние оболочки не видят ваших псевдонимов и локальных переменных** — только экспортные переменные.
- Если нужно передать функцию в дочернюю оболочку, можно использовать `export -f имя_функции`, но это редко требуется и может быть небезопасно.

Таким образом, **подоболочка — это «лёгкая» изоляция внутри одного процесса**, а **дочерняя оболочка — полноценный новый процесс с ограниченным наследованием**.
