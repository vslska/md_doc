
# Конвеер
## Ввод, вывод и каналы (Pipes) в Linux

> Конвейер (`|`) соединяет **stdout** одной команды с **stdin** другой.  
> Это основа эффективной работы в терминале: *цепочка простых утилит → мощный результат*.

---

##  Три стандартных потока

| Поток | Назначение                     | По умолчанию      |
|-------|--------------------------------|-------------------|
| `stdin`  | стандартный ввод         | клавиатура (`0`)  |
| `stdout` | стандартный вывод        | терминал (`1`)    |
| `stderr` | сообщения об ошибках     | терминал (`2`)    |

> ✅ `команда1 | команда2` — `stdout` `команды1` → `stdin` `команды2`  
> 	❌ Ошибки (`stderr`) **не передаются** по конвейеру, если не перенаправить (`2>&1`) (подробнее: [[2.1.  stderr и перенаправление потоков]])

## Ключевые утилиты для обработки потоков

| Команда         | Назначение                               | Примеры                                                                                                                  |
| --------------- | ---------------------------------------- | ------------------------------------------------------------------------------------------------------------------------ |
| `wc`            | Подсчёт строк, слов, байт                | `wc -l animals.txt` → 7<br>`cut -f3 animals.txt \| wc -l`                                                                |
| `head` / `tail` | Первые / последние строки                | `head -n3 animals.txt`<br>`tail -n1` — последняя строка                                                                  |
| `cut`           | Извлечение полей или символов            | `cut -f2 animals.txt` — 2-е поле (книги)<br>`cut -d: -f1 /etc/passwd` — имена пользователей<br>`cut -c1-5` — символы 1–5 |
| `grep`          | Фильтрация по шаблону                    | `grep Nutshell animals.txt`<br>`grep -v 2005` — всё, **кроме** 2005<br>`grep -w "robin"` — только целое слово            |
| `sort`          | Сортировка строк                         | `sort animals.txt` — алфавит<br>`sort -n -k3` — числовая по 3-му полю<br>`sort -r` — в обратном порядке                  |
| `uniq`          | Удаление/подсчёт **соседних** дубликатов | `sort file \| uniq` — уникальные строки<br>`... \| uniq -c` — с количеством<br>`... \| uniq -d` — только дубликаты       |

## Часто используемые конвейеры

| Задача                                        | Команда                                                                                                                                              |
| --------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------- |
| Год самой новой книги                         | `cut -f3 animals.txt \| sort -nr \| head -n1` → `2014`                                                                                               |
| Список пользователей (отсортированный)        | `cut -d: -f1 /etc/passwd \| sort`                                                                                                                    |
| Есть ли пользователь `brltty`?                | `grep -qw brltty /etc/passwd && echo "yes"`                                                                                                          |
| Самая частая оценка в `grades`                | `cut -f1 grades \| sort \| uniq -c \| sort -nr \| head -n1 \| cut -d' ' -f2`                                                                         |
| Найти дубликаты JPEG                          | `md5sum *.jpg \| sort \| uniq -w32 -d` *(только дубли!)*<br>или подробно:<br>`md5sum *.jpg \| sort \| uniq -c -w32 \| sort -nr \| grep -v "^\s*1 "`  |
| Какие файлы имеют контрольную сумму `abc...`? | `md5sum *.jpg \| grep "abc..." \| cut -d' ' -f2-`                                                                                                    |

> 💡 `uniq` работает **только с соседними** дубликатами → всегда предваряйте `sort`.

```bash
$ md5sum *.jpg | cut -c1-32 | sort | uniq -c | sort -nr
```

##  Полезные флаги и трюки

| Флаг | Команда | Действие |
|------|---------|----------|
| `-d` | `cut`, `sort` | Указать разделитель (`-d:` для `/etc/passwd`) |
| `-fN` | `cut` | Поле №N (например, `-f3` — третье поле) |
| `-w` | `grep` | Только целые слова |
| `-q` | `grep` | «тихий» режим — только код возврата (для `if`) |
| `-w32` | `uniq` | Сравнивать только первые 32 символа (для `md5sum`) |
| `2>&1` | в конвейере | Объединить `stderr` с `stdout`: `ls /nope 2>&1 \| grep "No"` |

---

## Советы

- Перед `rm`, `mv`, `> file` — **проверяйте конвейер через `echo` или `head`**
- Для работы с CSV/TAB: `cut` → быстро, но хрупко; `awk` или `csvkit` — надёжнее
- `|&` в bash = `2>&1 |` (сокращение для объединения stdout+stderr в конвейер)
