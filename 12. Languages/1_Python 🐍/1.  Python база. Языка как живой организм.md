## 1. Ключевые понятия

| Термин | Что это | Аналогия | Зачем знать | ⚠️ Частая ошибка |
|--------|---------|----------|-------------|------------------|
| **Интерпретатор** | Программа, которая _выполняет_ Python-код построчно | Устный переводчик на переговорах: слышит → думает → говорит | Без него — код «мертв». Ты выбираешь: системный, `venv`, `conda`. | Запускать через `python3 script.py`, а не `./script.py` без `#!/usr/bin/env python3` |
| **REPL** | Read-Eval-Print Loop — интерактивная консоль | Лабораторный микроскоп: проверяешь идеи мгновенно | Отладка, эксперименты, обучение | Писать в REPL многострочные `if`/`def` без отступов → SyntaxError |
| **Переменная** | Именованная **ссылка** на объект в памяти | Ярлык на ручке: «Маша», но ручка может быть у всех | Позволяет _переиспользовать_ данные. **Важно:** `a = 5; b = a` → обе ссылаются на `5` | `a = [1,2]; b = a; b.append(3)` → `a` тоже изменился! |
| **Тип данных** | Категория объекта: `int`, `str`, `list`, `bool`, `NoneType`… | Форма коробки: книга (immutable), бутылка (можно налить), воздушный шар (можно лопнуть) | От типа зависит, что можно делать: `"5" + 1` → `TypeError` | Смешивать `==` и `is` (`"a" is "a"` → True, но `"ab" is "a"+"b"` → False!) |
| **Объект** | Всё в Python — объект (даже `42`, `True`, функции!) | Любой предмет в мире имеет свойства и умеет что-то | Понимание методов, атрибутов, `id()`, `type()`, `dir()` | Думать, что `int` — «просто число», а не полноценный объект |

## 2. Типы данных

| Тип | Immutable? | Примеры | Важные методы | Лайфхак |
|-----|------------|---------|----------------|---------|
| `int` | ✅ | `42`, `-7`, `0x1F` | `bit_length()`, `to_bytes()` | `1_000_000` — можно ставить `_` для читаемости |
| `float` | ✅ | `3.14`, `1e-5` | `is_integer()`, `hex()` | `0.1 + 0.2 != 0.3` → используй `decimal.Decimal` для денег |
| `bool` | ✅ | `True`, `False` | — | `bool([])`, `bool("")`, `bool(0)` → `False` (falsy values) |
| `str` | ✅ | `"Привет!"`, `'🐍'`, `"""много\nстрок"""` | `.split()`, `.join()`, `.format()`, `f""` | f-строки — быстрее и чище: `f"{name=}"` → `"name='Алёна'"` |
| `NoneType` | ✅ | `None` | — | `if x is None:` — **не** `if x == None:` |
| `list` | ❌ | `[1, "a", []]` | `.append()`, `.pop()`, `.sort()` | `[0] * 5` → `[0, 0, 0, 0, 0]`, но `[[0]] * 3` → три ссылки на один список! |
| `tuple` | ✅ | `(1, "a")`, `(42,)` | `.count()`, `.index()` | Кортежи — для данных, которые **не должны меняться** (координаты, ключи) |
| `dict` | ❌ | `{"name": "Dima", "age": 30}` | `.get()`, `.items()`, `.setdefault()` | `d.get("key", default)` вместо `d["key"]` → не упадёт |
| `set` | ❌ | `{1, 2, 3}`, `set("abc")` | `.union()`, `.intersection()`, `.add()` | Быстро проверять принадлежность: `x in big_set` → O(1) |

> **Immutable ≠ «неизменяемый» в бытовом смысле** — нельзя *изменить содержимое*, но можно *переприсвоить переменную*:
> ```Pytnon
> s = "привет"
> s = s.upper()  # ✅ новая строка, старая — в мусорку
> ```

##  3. Базовые конструкции 

| Конструкция | Что делает | Синтаксис | Ловушка |
|-------------|------------|-----------|---------|
| **`import`** | Подключает модуль | `import math`, `from os import path`, `import numpy as np` | `from module import *` → засоряет пространство имён (избегай) |
| **`def`** | Определяет функцию | `def greet(name): return f"Hi, {name}"` | Аргументы по умолчанию — **не пересоздаются**: `def f(x=[])` → опасно! |
| **`if` / `elif` / `else`** | Ветвление | `if x > 0: ... elif x < 0: ... else: ...` | Проверяй порядок: первое истинное условие — выполняется, остальные — нет |
| **`for`** | Итерация по **итерируемому** (список, строка, генератор…) | `for i in range(3): ...` | `for i in list:` → `i` — копия значения, не ссылка на элемент! |
| **`while`** | Повтор, пока условие `True` | `while attempts > 0: ...` | Забыть изменить условие → бесконечный цикл (Ctrl+C!) |
| **`try` / `except`** | Обработка исключений | `try: risky(); except ValueError as e: ...` | Ловить `except:` **без типа** — плохо: скроешь `KeyboardInterrupt` |

## 4. Мини-шпаргалка: как проверить?

| Задача                          | Как проверить в REPL | Чтодает                                                                             |
| ------------------------------- | -------------------- | ----------------------------------------------------------------------------------- |
| Тип переменной                  | `type(x)`            | Показывает класс объекта (например, `<class 'int'>`).                               |
| ID объекта (уникальный адрес)   | `id(x)`              | \|Позволяет понять, ссылаются ли две переменные на один и тот же объект в памяти.\| |
| Всё, что умеет объект           | `dir(x)`             |                                                                                     |
| Помощь по функции/модулю        | `help(str.split)`    |                                                                                     |
| «Правда ли это?» (falsy/truthy) | `bool(x)`            |                                                                                     |
