Для ручной загрузки из консоли GRUB (когда разделы на **LVM** и **XFS**), следуйте этой последовательности команд.

### 1. Подготовка модулей

Сначала нужно загрузить поддержку LVM и файловой системы XFS, иначе GRUB не увидит содержимое разделов:

```bash
insmod lvm
insmod xfs
```
## 2. Поиск и установка Root-раздела

Посмотрите список доступных разделов: `ls`. LVM-тома будут выглядеть как `(lvm/volumegroup-logicalvolume)`.

1. **Установите root** для GRUB (где лежат ядра):
```
set root=(hd0,2)
```
 
 _Если `/` находится внутри LVM, указывайте путь к LVM-тому, например: `set root=(lvm/vg0-lv_root)`._

2. **Проверьте содержимое**, чтобы убедиться, что видите ядра:
``` 
ls /
```

## 3.  Загрузка старого ядра (Old)

Используйте клавишу **Tab** для автодополнения имен файлов.

1. **Загрузите ядро** (замените версию на вашу актуальную из списка `ls`):
```bash
linux vmlinuz-OLD_VERSION root=/dev/mapper/ВАША_ГРУППА-ВАШ_ТОМ ro init=/bin/bash
```

_Параметр `init=/bin/bash` нужен, чтобы сразу попасть в консоль и исправить ошибку с `umask`, не доходя до логина._

2. **Загрузите initrd** (версия должна строго совпадать с vmlinuz)
```bash
initrd /initrd.img-OLD_VERSION
```
## 4. Запуск
```bash
boot
```


---

## Что сделать сразу после загрузки:

Система загрузится в bash с правами root.

1. **Перемонтируйте диск на запись**:
```
mount -o remount,rw /
```

2. **Исправьте ошибку umask**:
```
nano /etc/login.defs
```

Найдите `umask` и исправьте на заглавные: **`UMASK 022`**. ( если 027 , то просто UMASK заместо umask)
1. **UMASK 022 (Стандарт):** Новые файлы доступны для чтения всем. Это обычное поведение для большинства Linux-систем.
2. **UMASK 027 (Безопасный):** Новые файлы доступны вам (полностью) и вашей группе (чтение), а **остальным пользователям доступ запрещен совсем**


3. **Восстановите GRUB**:
```
update-grub
```

(или `grub-mkconfig -o /boot/grub/grub.cfg`)
