
# Получение root-доступа через init в GRUB  
  
> **Внимание:** Этот метод позволяет получить полный root-доступ без пароля. Использовать только на своих системах или с разрешения администратора. Физический доступ к машине обязателен.  
  
---  
  
## 1. Загрузка через GRUB  
  
1. Перезагрузите систему и в меню GRUB выберите нужное ядро (например, старое, если новое не работает).    
2. Нажмите `e` для редактирования выбранного ядра.  
  
---  
  
## 2. Найдите строку `linux`  
  
Пример строки:  
  
```bash  
linux /boot/vmlinuz-<версия_ядра> root=/dev/mapper/<VG-LV> ro
```
- `<версия_ядра>` — версия ядра, которое вы хотите загрузить.
    
- `<VG-LV>` — логический том, который используется как root. 

---

## 3. Укажите init и initrd

В конец строки `linux` добавьте:

`init=/bin/bash`

Пример:

```
linux /boot/vmlinuz-5.15.0-70-generic root=/dev/mapper/astra-root ro init=/bin/bash
```

Убедитесь, что в GRUB также указан initrd:

```
initrd /boot/initrd.img-5.15.0-70-generic
```

---


## 4. Загрузка

- Нажмите `Ctrl+X` или `F10` для загрузки с редактированными параметрами.
    
- Система загрузится сразу в **root shell** (`bash-5.x#`), без запроса пароля.

---

## 5. Перевод файловой системы в режим записи

По умолчанию `/` монтируется как read-only. Сделайте так:

```bash
mount -o remount,rw /
```

Проверьте:

```bash
whoami  
```
> Должно вывести: root

---

## 6. Исправление PAM / паролей (опционально)

Если цель — починка PAM или root-пароля:

### Переустановка PAM  
```bash
apt install --reinstall libpam-modules libpam-modules-bin libpam-runtime  
```
  
### Для Astra SE (SELinux)  
```bash
restorecon -Rv /  
```
  
### Сброс пароля root  
```bash
passwd root
```

---

## 7. Перезагрузка в обычный режим

```bash
exec /sbin/init  
```
### или  
```bash
reboot -f
```

Теперь система загрузится стандартным образом, а root-доступ восстановлен.

---

## ⚠ Важные замечания по безопасности

- Любой, кто имеет **физический доступ** к машине и знает GRUB, может использовать этот метод.
    
- Чтобы защититься:
    
    - Установите пароль на GRUB (`set superusers` + `password_pbkdf2`)
        
    - Используйте шифрование root-диска (LUKS)
        
    - BIOS/UEFI пароль и Secure Boot
        
- Init-shell минует все проверки PAM и паролей.
