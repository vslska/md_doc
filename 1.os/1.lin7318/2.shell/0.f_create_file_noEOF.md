# Создание файлов в Fish Shell (вместо EOF)

В оболочке `fish` стандартный синтаксис `<< EOF` (Heredoc) не поддерживается. Вместо него используются другие подходы.

## 1. Простой многострочный echo (Рекомендуемый)
В `fish` можно просто переносить строку внутри кавычек. Текст сохранится ровно так, как вы его ввели.

```bash
echo "[app]
name = MyApp
port = 8080
log_level = info" > app.conf
```

## 2. Использование printf
Более строгий способ, если нужно точно контролировать переносы строк (`\n`) или табуляцию:

```bash
printf "[app]\nname = MyApp\nport = 8080\n" > app.conf
```

## 3. Вызов Bash (если нужен именно EOF)
Если у вас есть готовый кусок текста с `EOF` и вы не хотите его переделывать, пробросьте команду через Bash:

```bash
bash -c 'cat << EOF > app.conf
[app]
name = MyApp
EOF'
```

## 4. Команда `string collect`
В новых версиях fish можно использовать встроенные инструменты для обработки строк:

```bash
begin
    echo "[app]"
    echo "name = MyApp"
end | string collect > app.conf
```

## Важное замечание по отступам
В вашем примере были пробелы перед строками:
```text
    [app]
    name = MyApp
```
Если они попадут в `.conf` или `.yml` (Ansible), это может вызвать ошибки. Всегда пишите текст **вплотную к левому краю**, если только отступы не предусмотрены форматом файла.
