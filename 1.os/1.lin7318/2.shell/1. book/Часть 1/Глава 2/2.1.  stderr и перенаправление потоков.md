# stderr и перенаправление потоков

> В Linux у процесса **три стандартных потока**:  
> `stdin (0)` ← ввод, `stdout (1)` → обычный вывод, `stderr (2)` → ошибки.  
> **`>` и `|` работают только с `stdout`**, ошибки (`stderr`) продолжают идти в терминал.

---

## Основные операторы перенаправления

| Синтаксис  | Действие                                                                 |
| ---------- | ------------------------------------------------------------------------ |
| `> file`   | `stdout` → файл (перезапись)                                             |
| `>> file`  | `stdout` → файл (дописывание)                                            |
| `2> file`  | `stderr` → файл                                                          |
| `2>> file` | `stderr` → файл (дописывание)                                            |
| `&> file`  | `stdout` **и** `stderr` → файл                                           |
| `>&2`      | перенаправить `stdout` в `stderr` (часто в скриптах: `echo "Error" >&2`) |
| `\|&'      | `stdout + stderr` → пайп (эквивалент `2>&1 \|`)                          |

---

###  Примеры
```bash
# Только ошибки в файл:
cp nofile.txt dest 2> error.log

# Оба потока — в один файл:
cat ok.txt nofile.txt &> all.log

# Пайп с ошибками (bash):
ls /root /tmp 2>&1 | grep -i "permission\|tmp"

# Вывод ошибки в терминал, а нормальный — в файл:
command 2>&1 >out.txt   # ❌ stdout → out.txt, stderr → терминал  
command >out.txt 2>&1   # ✅ сначала stdout, потом stderr → out.txt
```

Порядок важен: `>file 2>&1` ≠ `2>&1 >file`

## Памятка: цифры потоков
|Число|Поток|Можно опустить?|
|---|---|---|
|`0`|stdin|`< file` вместо `0< file`|
|`1`|stdout|`> file` вместо `1> file`|
|`2`|stderr|`2>` — **нельзя** сократить|

>  `2>/dev/null` — «заглушить» ошибки:  
> `grep xyz *.log 2>/dev/null` — покажет только совпадения, без «No such file».
