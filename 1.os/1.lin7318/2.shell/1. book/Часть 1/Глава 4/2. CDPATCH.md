Командная оболочка предоставляет более общее решение для создания быстрых функций по переходу в нужный каталог, которое лишено этого недостатка. 
*Путь поиска команды `cd (cd search path)`* в свое время изменил подход к навигации по файловой системе Linux.

Было бы здорово, если бы мы могли сократить этот путь до нашего каталога, независимо от того, где мы находитесь в файловой системе.

  Путь поиска команды `cd` работает так же, как путь `$PATH`, но вместо команд он находит подкаталоги.
Например, мы часто пользуемся  каталогов `Photos`, который находится в `$HOME/Family/Memories`,  и нам нужно в него попасть с любого каталога, для этого мы настроим переменную оболочки `СDPATH`. которая имеет тот же формат, что и `PATH` — список каталогов, разделенных двоеточиями:
( В данном примере, помимо, мы добавим ещё 3 каталога)
```bash
export CDPATH=$HOME:$HOME/Projects:$HOME/Family/Memories:/usr/local
```
  
Печатаем:
```bash
$ cd Photos
```

Тогда команда cd проверит наличие следующих каталогов по порядку:
1. Подкаталог Photos в текущем каталоге
2. $HOME/Photos
3. $HOME/Projects/Photos
4. $HOME/Family/Memories/Photos
5. /usr/local/Photos

В нашем случае `cd` с четвертой попытки успешно меняет каталог на `$HOME/Family/Memories/Photos.` Если два каталога в `$CDPATH` имеют подкаталог с именем `Photos`, переход произойдет в тот из них, который стоит раньше в списке.
