### Способ #9. Фоновое выполнение команды

По умолчанию команды запускаются в **foreground** (переднем плане) — вы ждёте их завершения. Но можно запустить команду в **background** (фоне), чтобы сразу вернуть управление оболочке.

#### Как запустить в фоне?
Добавьте `&` в конец команды:
```bash
$ sleep 60 &
[1] 12345
$
```

- `[1]` — номер задания (job ID)
- `12345` — PID процесса

Команда выполняется в фоне, а вы можете продолжать работать.

>  **Job ≠ Process**:  
> Задание (job) — это концепция оболочки. Оно может включать несколько процессов (например, весь конвейер `cmd1 | cmd2 | cmd3` — одно задание).

#### Управление заданиями: ключевые команды
| Команда   | Действие                                             |
| --------- | ---------------------------------------------------- |
| `jobs`    | Показать все задания в текущей оболочке              |
| `fg [%n]` | Перевести задание на передний план                   |
| `bg [%n]` | Возобновить приостановленное задание в фоне          |
| `kill %n` | Завершить задание по номеру                          |
| `Ctrl-Z`  | Приостановить текущее задание (foreground → stopped) |
Примеры:
```bash
# Запустить в фоне
$ sleep 100 &

# Посмотреть задания
$ jobs
[1]+ Running   sleep 100 &

# Перевести на передний план
$ fg %1

# Или приостановить (Ctrl-Z), потом отправить в фон:
^Z
[1]+ Stopped   sleep 100
$ bg %1
```

> **Важно**:
> - `%n` — номер задания (не PID!)
> - Без `%` — действует на **текущее** задание (`fg`, `bg`, `kill`)

#### Ввод/вывод в фоне

- **stdout/stderr** фоновой команды всё ещё выводятся в терминал — это может "засорить" экран.
- **stdin** недоступен: если команда попытается прочитать ввод — она **автоматически приостанавливается**.
##### Решение:
Перенаправляйте ввод/вывод явно:
```bash
# Безопасный фон с логированием
tar -czf backup.tar.gz /home/user > /tmp/backup.log 2>&1 &

# Или подавите вывод полностью
long_task > /dev/null 2>&1 &
```
---
#### Практические сценарии для администратора

1. **Длительные задачи без блокировки терминала**
```bash
rsync -av /data/ user@remote:/backup/ > /var/log/rsync.log 2>&1 &
```

2. **Параллельный запуск нескольких задач**
```bash
task1 & task2 & task3 &
wait  # дождаться завершения всех фоновых заданий
```
3.  **Работа с редактором без перезапуска**
    
    - Открыли файл в `vim`
    - Нужно проверить что-то в терминале? Нажмите `Ctrl-Z`
    - Выполните команды
    - Вернитесь в редактор: `fg`
    
    → Экономит время и сохраняет контекст.

4. **Условные цепочки в фоне**
```bash
make && make install && systemctl restart myapp &
```

> Если `make` упадёт — остальные команды не выполнятся.

#### Советы и лайфхаки

- **`wait`** — дождаться завершения всех фоновых заданий в скрипте.
- **`disown`** — отвязать задание от оболочки, чтобы оно не завершилось при выходе:
```bash
long_task &
disown %1
exit  # задание продолжит работать
```

> - **Избегайте интерактивных команд в фоне** (`nano`, `less`, `read` и т.д.) — они зависнут.
#### Итог

Фоновое выполнение — не просто «удобство», а **инструмент эффективной работы**:

- Не ждите завершения долгих операций.
- Управляйте несколькими задачами в одной сессии.
- Сохраняйте контекст (редактор, каталог, переменные).

Но помните:
> **Фон — не демон.**  
> При закрытии терминала (без `disown` или `nohup`) фоновые задания получат сигнал `SIGHUP` и завершатся.

Для truly долгих задач используйте `nohup`, `screen` или `tmux`.
