### Создание `CDPATH` автоматически

**Цель**:  
Создать переменную `CDPATH`, которая позволяет переходить в подкаталоги домашней папки **без указания полного пути**:
```bash
cd Work      # вместо cd ~/Work
cd Linux     # вместо cd ~/Linux
```

Для этого нужно собрать строку вида:  
`PATH=$HOME:$HOME/Work:$HOME/Family:$HOME/Finances:$HOME/Linux:$HOME/Music:..`

#### Пошаговое построение однострочника

1. **Получить список подкаталогов в `$HOME`**
```bash
(cd && ls -d */)
```
> → `(cd ...)` гарантирует, что текущий каталог не изменится.

2. **Добавить `$HOME/` к каждому имени**  
   Используем `sed` с нестандартным разделителем (`@`), чтобы избежать экранирования `/`:
   ```bash
   sed 's@^@$HOME/@'
   ```

3. **Убрать завершающий `/`**  
   ```bash
   sed 's@/$@@'
   ```

4. **Объединить всё в одну строку через пробелы**  
   Подстановка команды сама создаёт подоболочку:
   ```bash
   echo $(cd && ls -d */ | sed -e 's@^@$HOME/@' -e 's@/$@@')
   ```

5. **Добавить `$HOME` в начало и `..` в конец**  
   ```bash
   echo '$HOME' $(...) ..
   ```

6. **Заменить пробелы на двоеточия**  
   ```bash
   tr ' ' ':'
   ```

7. **Добавить префикс `CDPATH=`**  
   ```bash
   echo "CDPATH=$(echo '$HOME' $(cd && ls -d */ | sed -e 's@^@$HOME/@' -e 's@/$@@') .. | tr ' ' ':')"
   ```

Или проще — сразу сформировать готовую строку для `.bashrc`:
```bash
echo 'CDPATH=$HOME:'$(cd && ls -d */ | sed -e 's@^@$HOME/@' -e 's@/$@@' | tr '\n' ':')'..'
```

---
###  Улучшенная версия (надёжнее)

Проблема оригинального подхода: `ls -d */` **ломается**, если в домашнем каталоге нет подкаталогов (ошибка). Также он не обрабатывает имена с пробелами корректно.

**Более надёжный способ — через glob и массивы (Bash):**
```bash
dirs=("$HOME" "$HOME"/ */ ..)
# Убираем слеши и формируем CDPATH
paths=()
for d in "${dirs[@]}"; do
    [[ -d "$d" ]] && paths+=("${d%/}")  # убираем завершающий /
done
IFS=:; echo "CDPATH=${paths[*]}"
```

Но если нужен именно **однострочник для терминала**, вот безопасный вариант:
```bash
( shopt -s nullglob; cd; printf '%s:' "$PWD" */ .. ) | sed 's/:\/$/:/' | sed 's/:$//'
```

→ `nullglob` гарантирует, что `*/` не останется как строка, если подкаталогов нет.

---

### Как использовать?

Добавьте в `~/.bashrc`:
```bash
export CDPATH=$(cd && echo -n "$HOME:"; ls -d */ 2>/dev/null | sed 's@/$@@' | sed "s@^@$HOME/@" | tr '\n' ':' ; echo "..")
```

Или проще — генерируйте один раз и вставляйте вручную:
```bash
echo "export CDPATH=\$(cd && echo -n '\$HOME:'; ls -d */ 2>/dev/null | sed 's@/$@@' | sed 's@^@\$HOME/@' | tr '\n' ':' ; echo '..')"
```

После перезагрузки `.bashrc`:
```bash
source ~/.bashrc
cd Music   # работает!
```

---

###  Важные нюансы

- **`CDPATH` влияет только на `cd`**, не на другие команды.
- Если имя каталога совпадает с именем в текущем каталоге — `cd` может выбрать не тот!  
  → Чтобы гарантированно перейти в текущий каталог: `cd ./имя`
- Не все оболочки поддерживают `CDPATH` (например, `fish` — нет). Работает в `bash`, `zsh`.

---

### Альтернатива: псевдонимы или функции

Если подкаталогов немного, проще создать псевдонимы:
```bash
alias cdl='cd ~/Linux'
alias cdw='cd ~/Work'
```

Или функцию:
```bash
c() { cd ~/"$1"; }
# c Work → cd ~/Work
```

→ Это читаемее и безопаснее, чем динамический `CDPATH`.

---

### Итог

Автоматическая генерация `CDPATH` — отличный пример того, как:
- использовать подоболочки для изоляции,
- комбинировать `sed`, `tr`, подстановку команд,
- решать рутинную задачу одной строкой.

Но помните:  
> **Удобство ≠ надёжность.**  
> Для production-окружений или сложных имён файлов лучше использовать явные пути или функции.
