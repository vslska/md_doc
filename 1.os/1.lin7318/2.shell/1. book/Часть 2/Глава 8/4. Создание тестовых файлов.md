### Создание тестовых файлов: однострочники для генерации данных

Часто при разработке или отладке нужно **много файлов** — с разным содержимым, именами или размерами. Вместо ручного создания можно использовать мощь оболочки.
#### 1. Генерация случайных текстовых файлов

**Цель**: создать 1000 файлов со случайным количеством случайных слов.

**Инструменты**:
- `shuf` — перемешивает строки и выбирает N случайных
- `pwgen` — генерирует случайные имена (`apt install pwgen`, если нет)
- `$RANDOM` — встроенная переменная Bash (0–32767)

**Однострочник**:
```bash
yes 'shuf -n $RANDOM /usr/share/dict/words > $(pwgen -N1 10).txt' | head -1000 | bash
```

>  Почему `>` вместо `-o`?  
> Потому что `-o` требует имени файла **до выполнения**, а нам нужно, чтобы `pwgen` вызывался **каждый раз заново**. Через перенаправление `>` это работает.

**Как это работает**:
- `yes '...'` → бесконечно повторяет строку
- `head -1000` → берёт первые 1000 копий
- `| bash` → выполняет каждую как команду
- `$RANDOM` и `$(pwgen ...)` вычисляются **при каждом запуске**

> **Ловушка**:  
> Если написать `shuf -n $RANDOM -o $(pwgen...).txt ...`, то `$(pwgen...)` вычислится **один раз** до запуска `yes` → все файлы получат одно имя!  
> Поэтому используйте перенаправление `>` внутри строки.

#### 2. Генерация случайных изображений (если установлен ImageMagick)

```bash
yes 'convert -size 8x8 xc: +noise Random -scale 100x100 $(pwgen -N1 10).png' | head -1000 | bash
```

→ Создаёт 1000 PNG-файлов с шумом. Полезно для тестирования загрузчиков изображений, сканеров и т.д.

>  Установка:  
> ```bash
> sudo apt install imagemagick    # Debian/Ubuntu
> ```

#### 3. Создание пустых файлов

##### Простой способ — нумерованные имена:
```bash
touch file{0001..1000}.txt
```

##### Случайные имена из словаря:
```bash
grep '^[a-z]\+$' /usr/share/dict/words | shuf | head -1000 | xargs touch
```

>  Почему `grep '^[a-z]\+$'`?  
> Чтобы исключить слова с апострофами (`don't`), заглавными буквами или цифрами — такие имена могут сломать команды без кавычек.

>  **Проблема `xargs touch`**:  
> Если имя файла содержит пробел — команда упадёт.  
> **Безопасная версия** (Bash):
> ```bash
> mapfile -t words < <(grep '^[a-z]\+$' /usr/share/dict/words | shuf | head -1000)
> for w in "${words[@]}"; do touch -- "$w"; done
> ```

### Профессиональные фишки

#### Dry-run перед массовым созданием
Всегда проверяйте, что будет создано:
```bash
yes 'echo shuf -n $RANDOM ... > $(pwgen -N1 10).txt' | head -5
```
→ Увидите 5 примеров команд **без выполнения**.

#### Очистка после тестов
```bash
# Удалить все .txt в каталоге
rm *.txt

# Или безопаснее — через find
find . -name "*.txt" -delete
```

#### Генерация файлов заданного размера
```bash
# 10 файлов по 1 МБ
for i in {1..10}; do dd if=/dev/urandom of=file$i.bin bs=1M count=1; done
```

### Когда НЕ использовать такие однострочники?

- Если нужны **точные данные** (например, JSON с определённой структурой) → пишите скрипт на Python.
- Если важна **воспроизводимость** → зафиксируйте seed (`shuf -r --random-source=/dev/urandom` не поможет, лучше использовать `sort -R` с фиксированным `LC_ALL=C`).
- Если файлов **миллионы** → лучше использовать `parallel` или скрипт с прогресс-баром.

---

### Итог

Эти однострочники — не «магия», а **системное применение**:
- генераторов (`shuf`, `pwgen`),
- потоковой обработки (`yes`, `head`, `xargs`),
- динамического выполнения (`| bash`).

Они идеальны для:
- быстрой генерации тестовых данных,
- нагрузочного тестирования,
- демонстрации возможностей терминала.

Но помни:  
> **Однострочник — инструмент для разовых задач.**  
> Для повторяющегося использования — выносите логику в скрипт.
