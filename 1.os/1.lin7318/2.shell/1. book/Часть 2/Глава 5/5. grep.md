```bash
$  grep  Cisco animals.txt      # Вывести строки, содержащие
donkey  Cisco IOS in a Nutshell 2005    Boney, James
```

```bash
$  grep -w Cisco animals.txt   # Искать точное совпадение, дэш ворд ммааа бро.
donkey  Cisco IOS in a Nutshell 2005    Boney, James

$  grep  -i cisco animals.txt  # Дэш Ай, игнор регистра букв

$  grep  -l Cisco *  # В каком файлесодержаться совпадающие  строки
animals.txt  
```

`grep` становится, когда используем [[Регулярные выражения]]


### Примеры `grep` с регулярными выражениями
```bash
$  grep '^[A-Z]' myfile  # Найти совпадение всех строк, начинающихся с заглавной буквы

$  grep -v '^$' myfile     # Найти совпадение со всеми непустыми строками (то есть совпадение с пустыми строками и их пропуск благодаря параметру -v)

$  grep 'cookie\|cake' myfile  # Найти все строки, содержащие либо cookie, либо cake

$  grep '.....' myfile     # Найти все строки длиной не менее пяти символов

$  grep '<.*>' page.html   # Найти все строки, в которых символ «меньше» появляется перед символом «больше»
```

Иногда регулярные выражения  приводят к неожиданным результатам. Например, хотим *найти в файле*  *две строки*, содержащие букву '*w* ', В след команде неправльныйрузельтат, т.к. *точка* *(dot)*, регулярное выражение "*любой символ*" 

```bash
$  grep w. frost
Whose woods these are I think I know.
He will not see me stopping here
To watch his woods fill up with snow.
```

Чтобы обойти, экранируем символ:
```bash
$  grep 'w\.' frost
Whose woods these are I think I know.
To watch his woods fill up with snow.
```

## Факты
> - Название grep является сокращением от «получить регулярное выражение и распечатать» (get regular expression and print).C
