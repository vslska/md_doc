## Архитектура выполнения `awk`

### Алгоритм обработки входного потока
```
Для каждой входной записи (строки по умолчанию):
1. Разделить запись на поля по разделителю FS (по умолчанию: пробел/таб)
2. Присвоить:
   - $0 = вся запись (исходная строка)
   - $1, $2, ... $NF = значения полей
   - NF = количество полей
   - NR = номер записи в общем потоке
   - FNR = номер записи в текущем файле
3. Для каждого правила "паттерн { действие }":
   - Если паттерн совпадает → выполнить действие
4. После обработки всех записей → выполнить блок END (если задан)
```

### Базовый синтаксис программы
```bash
# Общая форма
awk 'паттерн { действие }' файл

# Множественные правила
awk '
    BEGIN { инициализация }      # Выполняется один раз перед обработкой
    паттерн1 { действие1 }        # Выполняется при совпадении паттерна1
    паттерн2 { действие2 }        # Выполняется при совпадении паттерна2
    END { финализация }           # Выполняется один раз после обработки
' файл
```

>[!NOTE] Исполняемая форма
>```bash
># Однострочник (в кавычках)
awk '/error/ { print $1, $3 }' /var/log/syslog
>
># Скрипт-файл (без кавычек, с #!/usr/bin/awk -f)
># script.awk
BEGIN { FS=":" }
/root/ { print $1, $7 }
>
># Запуск:
awk -f script.awk /etc/passwd
>```

## Встроенные переменные: справочник

### Переменные управления вводом/выводом

| Переменная | Значение по умолчанию | Описание                                                 | Пример изменения                      |
| ---------- | --------------------- | -------------------------------------------------------- | ------------------------------------- |
| `FS`       | `" "` (пробел/таб)    | Разделитель полей **входа** (Field Separator)            | `-F':'` или `BEGIN { FS=":" }`        |
| `OFS`      | `" "`                 | Разделитель полей **вывода** (Output Field Separator)    | `BEGIN { OFS="\t" }`                  |
| `RS`       | `"\n"`                | Разделитель записей **входа** (Record Separator)         | `BEGIN { RS="" }` (параграфный режим) |
| `ORS`      | `"\n"`                | Разделитель записей **вывода** (Output Record Separator) | `BEGIN { ORS="\r\n" }`                |
| `SUBSEP`   | `"\034"`              | Разделитель индексов в многомерных массивах              | Обычно не меняется                    |
### Переменные данных (только для чтения)

|Переменная|Тип|Описание|Пример|
|---|---|---|---|
|`$0`|string|Вся текущая запись (исходная строка)|`print $0`|
|`$1`..`$NF`|string|Значения полей (нумерация с 1)|`print $1, $3`|
|`NF`|int|Количество полей в текущей записи|`if (NF > 5) { ... }`|
|`NR`|int|Номер текущей записи во **всех** входных файлах|`NR == 1 { print "First line" }`|
|`FNR`|int|Номер текущей записи в **текущем** файле|`FNR == 1 { print "New file:", FILENAME }`|
|`FILENAME`|string|Имя текущего обрабатываемого файла|`print FILENAME ":" NR`|
|`ARGC`, `ARGV`|int/array|Аргументы командной строки (как в C)|`for (i=1; i<ARGC; i++) print ARGV[i]`|

### Пример: изменение `FS` и `OFS`
```bash
# Парсинг /etc/passwd (разделитель ":")
awk 'BEGIN { FS=":"; OFS="\t" } { print $1, $7 }' /etc/passwd

# Вывод:
# root    /bin/bash
# daemon  /usr/sbin/nologin
# ...
```

>[!WARNING] `$0` перезаписывается при изменении полей
> Если вы модифицируете `$1`, `$2` и т.д., `$0` автоматически пересобирается с использованием `OFS`:
>```bash
> # Исходная строка: "foo:bar:baz"
BEGIN { FS=":"; OFS="|" }
{ $2 = "MODIFIED"; print $0 }
># Вывод: "foo|MODIFIED|baz" (обратите внимание на OFS)
>```

## Паттерны: типы и синтаксис

Паттерн определяет, **когда** выполнять действие. Если паттерн опущен → действие выполняется для каждой записи.

### Типы паттернов

|Тип|Синтаксис|Описание|Пример|
|---|---|---|---|
|**Регулярное выражение**|`/регекс/`|Совпадение с `$0` или полем|`/error/ { print }`|
|**Реляционное выражение**|`поле оператор значение`|Сравнение полей|`$3 > 100 { print $1 }`|
|**Составной паттерн**|`паттерн1 && паттерн2`|Логические операторы|`/error/ && $5 > 50 { ... }`|
|**Диапазон**|`паттерн1, паттерн2`|От первого совпадения до второго|`/START/,/END/ { print }`|
|**Блок инициализации**|`BEGIN`|Выполняется один раз **до** чтения записей|`BEGIN { print "Header" }`|
|**Блок завершения**|`END`|Выполняется один раз **после** всех записей|`END { print "Total:", NR }`|

### Примеры паттернов
```bash
# 1. Фильтрация по регулярному выражению (регекс в $0)
awk '/failed/ { print $1, $4 }' /var/log/auth.log

# 2. Фильтрация по значению поля (реляционное)
awk '$3 > 1000 { print $1, $3 }' processes.txt

# 3. Составной паттерн (И/ИЛИ)
awk '/error/ && $5 >= "ERROR" { count++ } END { print count }' app.log

# 4. Диапазон: от строки с "BEGIN" до строки с "END"
awk '/^BEGIN/,/^END/ { print }' config.txt

# 5. Исключение: все записи, НЕ содержащие паттерн
awk '!/debug/ { print }' verbose.log

# 6. Паттерн на конкретное поле (через ~ и !~)
awk '$4 ~ /^[0-9]{4}-[0-9]{2}-[0-9]{2}$/ { print }' data.csv
```

>[!NOTE] Операторы соответствия `~` и `!~`
>```bash
>$1 ~ /pattern/   # Поле $1 соответствует регексу
$2 !~ /pattern/  # Поле $2 НЕ соответствует регексу
>
># Эквивалентно:
/pattern/        # Регекс применяется к $0 (всей строке)
>```

## Действия: команды и управляющие конструкции

### Базовые команды

|Команда|Описание|Пример|
|---|---|---|
|`print`|Вывод полей/выражений (с `ORS`)|`print $1, $3`|
|`printf`|Форматированный вывод (как в C)|`printf "%s\t%d\n", $1, $3`|
|`next`|Пропустить остаток правил для текущей записи|`/^#/ { next }`|
|`exit [код]`|Немедленно завершить обработку|`NR > 100 { exit }`|
|`delete array[index]`|Удалить элемент массива|`delete seen[$1]`|

### Управляющие конструкции (как в C)
```bash
# Условный оператор
if ($3 > 100) {
    print $1, "HIGH"
} else {
    print $1, "LOW"
}

# Цикл for (по индексам массива)
for (key in array) {
    print key, array[key]
}

# Цикл for (числовой)
for (i = 1; i <= NF; i++) {
    printf "Field %d: %s\n", i, $i
}

# Цикл while / do-while
while (getline > 0) {
    total++
}

# Операторы управления потоком
break      # Выход из цикла
continue   # Переход к следующей итерации
```

### Пример: агрегация данных
```bash
# Подсчёт запросов по статус-кодам в access.log
awk '
    { status[$9]++ }
    END {
        for (code in status) {
            printf "%s: %d\n", code, status[code]
        }
    }
' /var/log/nginx/access.log
```

## Функции `awk`: справочник

### Строковые функции

|Функция|Описание|Пример|Результат|
|---|---|---|---|
|`length([s])`|Длина строки|`length($1)`|`5` для `"hello"`|
|`substr(s, start, [len])`|Подстрока|`substr($1, 1, 3)`|`"hel"` из `"hello"`|
|`index(s, t)`|Позиция подстроки|`index("hello", "ll")`|`3`|
|`match(s, regex)`|Поиск регекса, установка `RSTART`, `RLENGTH`|`match($0, /[0-9]+/)`|`RSTART=5, RLENGTH=2`|
|`sub(regex, repl, [target])`|Замена **первого** вхождения|`sub(/old/, "new", $0)`|Модифицирует `$0`|
|`gsub(regex, repl, [target])`|Замена **всех** вхождений|`gsub(/,/, ";", $0)`|Все запятые → точки с запятой|
|`split(s, array, [sep])`|Разбить строку в массив|`split($1, arr, ":")`|`arr[1]`, `arr[2]`, ...|
|`tolower(s)` / `toupper(s)`|Регистр|`toupper($1)`|`"HELLO"`|
|`sprintf(fmt, expr...)`|Форматирование в строку|`sprintf("%.2f", 3.14159)`|`"3.14"`|

### Математические функции
```bash
# Стандартные функции (как в C)
int(x)      # Целая часть
sqrt(x)     # Квадратный корень
log(x), exp(x)  # Логарифм, экспонента
sin(x), cos(x), atan2(y,x)  # Тригонометрия
rand()      # Случайное число [0,1)
srand([seed])  # Инициализация генератора

# Пример: вычисление среднего
awk '{ sum += $1; count++ } END { print "Avg:", sum/count }' numbers.txt
```

### Функции работы с временем (GNU awk)
```bash
# Конвертация и форматирование времени
systime()              # Текущее время как Unix-epoch
strftime(fmt, [epoch]) # Форматирование epoch → строка
mktime("YYYY MM DD HH MM SS")  # Строка → epoch

# Пример: добавить временную метку к выводу
awk '{ print strftime("%Y-%m-%d %H:%M:%S"), $0 }' input.log
```

>[!NOTE] GNU awk (gawk) расширения 
>Некоторые функции (`mktime`, `strftime`, `and()`, `or()`, `xor()`) доступны только в GNU awk. Проверьте версию:
>```bash
>awk --version  # или gawk --version
>```

## Массивы в `awk`

### Типы массивов

`awk` поддерживает только **ассоциативные массивы** (ключи — строки), но числовые индексы автоматически преобразуются в строки.
```bash
# Инициализация и доступ
array["key"] = "value"
print array["key"]

# Числовой индекс (преобразуется в строку)
array[1] = "first"
print array["1"]  # Выведет "first"
```

### Итерация по массиву
```bash
# Порядок итерации НЕ гарантирован (хеш-таблица)
for (key in array) {
    print key, array[key]
}

# Сортировка ключей (GNU awk: asorti)
asorti(array, sorted_keys)
for (i = 1; i <= length(sorted_keys); i++) {
    key = sorted_keys[i]
    print key, array[key]
}
```

### Многомерные массивы
```bash
# Синтаксис: массив[key1, key2, ...]
# Внутренне используется SUBSEP для объединения ключей
count[$1, $2]++

# Итерация:
for (combo in count) {
    split(combo, parts, SUBSEP)
    print parts[1], parts[2], count[combo]
}
```

### Пример: подсчёт уникальных значений
```bash
# Уникальные IP из access.log с количеством запросов
awk '
    { ip_count[$1]++ }
    END {
        for (ip in ip_count) {
            printf "%s: %d requests\n", ip, ip_count[ip]
        }
    }
' /var/log/nginx/access.log | sort -t: -k2 -nr
```

## Практические сценарии
### Сценарий 1: Парсинг логов (Nginx/Apache)
```bash
# Извлечь 404 ошибки с реферером и агентом
awk '
    $9 == 404 {
        # $7 = запрос, $11 = реферер, $12+ = user-agent
        gsub(/"/, "", $11)  # Убрать кавычки
        printf "%s %s %s\n", $7, $11, substr($0, index($0,$12))
    }
' /var/log/nginx/access.log

# Агрегация по статус-кодам с процентами
awk '
    { codes[$9]++; total++ }
    END {
        for (code in codes) {
            pct = (codes[code] / total) * 100
            printf "%s: %d (%.2f%%)\n", code, codes[code], pct
        }
    }
' /var/log/nginx/access.log
```
### Сценарий 2: Обработка CSV/TSV
```bash
# CSV с запятыми: выбрать колонки 1, 3, 5, отфильтровать по условию
awk '
    BEGIN { FS=","; OFS="," }
    NR == 1 { print; next }  # Заголовок
    $3 > 100 && $5 == "active" { print $1, $3, $5 }
' data.csv

# TSV: добавить вычисляемую колонку
awk '
    BEGIN { FS=OFS="\t" }
    NR == 1 { print $0, "total"; next }
    { print $0, $2 + $3 + $4 }
' report.tsv
```

### Сценарий 3: Анализ системных данных
```bash
# Парсинг вывода ps: процессы с потреблением памяти > 5%
ps aux | awk '
    NR == 1 { print; next }  # Заголовок
    $4 > 5.0 { printf "%-10s %-6s %-5s %s\n", $1, $2, $4"%", $11 }
'

# Анализ df: диски с заполнением > 80%
df -h | awk '
    NR == 1 { next }  # Пропустить заголовок
    {
        gsub(/%/, "", $5)  # Убрать % из колонки Use%
        if ($5 > 80) {
            printf "WARNING: %s used %d%% on %s\n", $6, $5, $1
        }
    }
'
```

### Сценарий 4: Генерация отчётов
```bash
# Группировка и суммирование: расходы по категориям
awk '
    BEGIN { FS=","; OFS="\t" }
    NR > 1 {
        gsub(/[[:space:]]+/, "", $2)  # Trim категории
        total[$2] += $3
    }
    END {
        print "Category", "Total"
        for (cat in total) {
            printf "%s\t%.2f\n", cat, total[cat]
        }
    }
' expenses.csv | column -t
```

## Отладка и типичные ошибки

### Частые проблемы и решения

|Ошибка|Причина|Решение|
|---|---|---|
|Поля не разделяются как ожидается|`FS` не установлен или неверный|Явно задать `BEGIN { FS=":" }` или `-F':'`|
|`$0` не обновляется после изменения полей|Ожидание, что `$0` останется исходным|Помнить: изменение `$1..$NF` пересобирает `$0` с `OFS`|
|Массив не сортируется при итерации|`for (key in array)` не гарантирует порядок|Использовать `asorti()` (gawk) или сортировать вывод внешним `sort`|
|Регекс не работает как в grep|Синтаксис регексов в awk — ERE (Extended)|Использовать `awk`-совместимые паттерны: `+`, `?`, `|
|`sub()`/`gsub()` не меняют исходное поле|Забыли, что функции модифицируют target|`sub(/old/, "new", $1)` меняет `$1`; без target — меняет `$0`|
|Числовое сравнение работает как строковое|Поле содержит нечисловые символы|Использовать `$3 + 0` для принудительного преобразования: `if (($3+0) > 100)`|

### Инструменты отладки
```bash
# Вывод отладочной информации
awk '
    { 
        printf "NR=%d FNR=%d NF=%d $0=[%s]\n", NR, FNR, NF, $0 > "/dev/stderr"
        for (i=1; i<=NF; i++) printf "  $%d=[%s]\n", i, $i > "/dev/stderr"
    }
' input.txt

# Проверка значения переменной
awk 'BEGIN { print "FS=[" FS "]"; FS=":"; print "New FS=[" FS "]" }'

# Тестирование регекса
echo "test123" | awk '/[0-9]+/ { print "Matched:", $0 }'
```

## Выводы

 **Архитектура `awk`**:

- Модель "паттерн → действие" с автоматической итерацией по записям
- Встроенные переменные (`FS`, `NF`, `NR`, `$0`..`$N`) предоставляют контекст без явного парсинга
- Блоки `BEGIN`/`END` позволяют инициализацию и финализацию вне основного цикла

 **Обработка данных**:

- Поля разделяются по `FS`, вывод форматируется через `OFS` и `printf`
- Регулярные выражения (ERE-синтаксис) применяются через `/паттерн/` или `~`/`!~`
- Агрегация через ассоциативные массивы: `count[key]++` — стандартный паттерн

 **Функции и расширения**:

- Строковые (`sub`, `gsub`, `split`, `substr`) и математические функции покрывают большинство задач трансформации
- `match()`, `RSTART`, `RLENGTH` — для сложного извлечения подстрок
- `strftime()`, `mktime()` (gawk) — для работы с временными метками

 **Практическое применение в DevOps**:

- Парсинг логов (Nginx, Apache, auth.log) с фильтрацией и агрегацией
- Обработка CSV/TSV: выбор колонок, фильтрация, вычисляемые поля
- Анализ системных утилит (`ps`, `df`, `netstat`) через конвейеры
- Генерация отчётов: группировка, суммирование, форматированный вывод

## Лабораторная работа: Анализ логов с `awk`

### Задания

#### Шаг 0: Подготовка тестовых данных
```bash
# Создать каталог лабы
mkdir -p ~/linux-labs/awk-analysis
cd ~/linux-labs/awk-analysis

# Сгенерировать тестовый access.log (100 строк)
cat > generate_log.sh << 'EOF'
#!/bin/bash
ips=("192.168.1.10" "10.0.0.5" "172.16.0.20" "192.168.1.15")
codes=(200 200 200 301 404 500)
paths=("/" "/api/users" "/static/main.css" "/admin" "/health")
for i in {1..100}; do
    ip=${ips[$((RANDOM % ${#ips[@]}))]}
    code=${codes[$((RANDOM % ${#codes[@]}))]}
    path=${paths[$((RANDOM % ${#paths[@]}))]}
    size=$((RANDOM % 5000 + 100))
    printf '%s - - [01/Jan/2026:12:00:%02d +0000] "GET %s HTTP/1.1" %d %d "-" "Mozilla/5.0"\n' \
        "$ip" "$((i % 60))" "$path" "$code" "$size"
done
EOF
chmod +x generate_log.sh
./generate_log.sh > access.log
```

#### Шаг 1: Базовая фильтрация
```bash
# 1.1 Вывести все записи с 404 статусом
awk '$9 == 404 { print }' access.log

# 1.2 Подсчитать количество запросов по каждому статус-коду
awk '{ codes[$9]++ } END { for (c in codes) print c, codes[c] }' access.log | sort -k2 -nr

# 1.3 Вывести уникальные IP с количеством запросов
awk '{ ips[$1]++ } END { for (ip in ips) printf "%s: %d\n", ip, ips[ip] }' access.log | sort -t: -k2 -nr
```

#### Шаг 2: Агрегация и отчёт
```bash
# 2.1 Создать скрипт report.awk
cat > report.awk << 'EOF'
BEGIN {
    FS = "\""
    OFS = "\t"
    print "IP", "Path", "Status", "Size", "Timestamp"
    print "----", "----", "------", "----", "---------"
}
{
    # Парсинг комбинированного формата: $1 = IP + ident + user, $2 = запрос, $3 = остальное
    split($1, meta, " ")
    ip = meta[1]
    
    # $2 = "GET /path HTTP/1.1"
    split($2, req, " ")
    path = req[2]
    
    # $3 = " status size "
    split($3, resp, " ")
    status = resp[2]
    size = resp[3]
    
    # Извлечь дату из $1 (после [)
    match($0, /\[([^]]+)\]/, dt)
    timestamp = dt[1]
    
    # Фильтр: только ошибки 4xx и 5xx
    if (status ~ /^[45][0-9][0-9]$/) {
        printf "%s\t%s\t%s\t%s\t%s\n", ip, path, status, size, timestamp
    }
}
END {
    print "----", "----", "------", "----", "---------"
    print "Total errors:", ERR_COUNT
}
EOF

# 2.2 Запустить отчёт
awk -f report.awk access.log
```
