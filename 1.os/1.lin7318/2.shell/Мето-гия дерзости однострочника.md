### Методология проектирования Unix One-Liners (2026) 
> Методология дерзости одностраничника.

### 1. Базовая анатомия конвейера

Конвейер строится слева направо. Данные текут через символ трубы `|` (pipe), который передает `STDOUT` (стандартный вывод) предыдущей команды на `STDIN` (стандартный ввод) следующей.

```bash
ГЕНЕРАТОР | ФИЛЬТР | ТРАНСФОРМАТОР | АГРЕГАТОР | ДЕЙСТВИЕ
```

| Этап              | Инструменты                       | Описание                                           |
| ----------------- | --------------------------------- | -------------------------------------------------- |
| **Генератор**     | `find`, `cat`, `ls`, `curl`, `ps` | Порождает поток текста.                            |
| **Фильтр**        | `grep`, `head`, `tail`, `uniq`    | Отсеивает немного нужные строки.                   |
| **Трансформатор** | `sed`, `awk`, `tr`, `cut`         | Изменяет структуру строки (столбцы, символы).      |
| **Агрегатор**     | `sort`, `wc`, `uniq -c`           | Группирует, считает или сортирует данные.          |
| **Действие**      | `xargs`, `parallel`, `tee`        | Выполняет финальную операцию или сохраняет в файл. |
### 2. «Золотая троица» инструментов

##### AWK (Король столбцов)

Если данные разделены пробелами или табуляцией, используйте `awk`.

- **Синтаксис:** `awk '{print $N}'` — где N это номер колонки.
- _Пример:_ Вывести только имена пользователей из системы:  
    `cat /etc/passwd | awk -F: '{print $1}'` (здесь `-F:` меняет разделитель на двоеточие).

##### SED (Мастер замены)

Идеален для правки текста «на лету».

- **Синтаксис:** `sed 's/что_ищем/на_что_меняем/g'`
- _Пример:_ Очистить лог от квадратных скобок:  
    `cat log.txt | sed 's/\[//g' | sed 's/\]//g'`

##### XARGS (Переход к действию)

Берет список строк и превращает их в аргументы для другой команды.

- _Пример:_ Найти все `.log` файлы и удалить их:  
    `find . -name "*.log" | xargs rm`

### 3. Пошаговая сборка «Дерзкого» однострочника

Допустим, задача: **Найти 5 самых «прожорливых» по памяти процессов и вывести их PID и Имя.**

**Шаг 1: Получаем данные (Генератор)**
```bash
ps aux
```

> _Проблема: Слишком много мусора._

**Шаг 2: Сортируем по 4-й колонке (физическая память)**
```bash
ps aux --sort=-%mem
```

**Шаг 3: Оставляем только нужные колонки (Трансформатор)**  

Мы хотим PID ($2) и Команду ($11).
```bash
ps aux --sort=-%mem | awk '{print $2, $11}'
```

**Шаг 4: Ограничиваем количество (Агрегатор)**  
Берем первые 5, пропуская заголовок (через `tail` или `sed`).
```bash
ps aux --sort=-%mem | awk '{print $2, $11}' | head -n 6
```

### 4. Продвинутые приемы
##### Перенаправление потоков (I/O Redirection)

- `> file` — перезаписать файл.
- `>> file` — добавить в конец файла.
- `2> /dev/null` — «Дерзкий» прием: отправить все сообщения об ошибках в черную дыру (в консоли будет только чистый результат).

##### Логические операторы (Цепочки)

- `cmd1 && cmd2` — выполнить `cmd2` только если `cmd1` завершилась успешно.
- `cmd1 || cmd2` — выполнить `cmd2` только если `cmd1` упала с ошибкой.

### 5. Практические упражнения для обучения

Попробуйте разобрать и выполнить эти команды в терминале:

1. **Посчитать количество файлов в текущей директории:**  
    `ls -1 | wc -l`
2. **Найти слово "Error" во всех логах и посчитать, сколько раз оно встретилось:**  
    `grep -ro "Error" /var/log | wc -l`
3. **Быстро узнать свой внешний IP адрес:**  
    `curl -s ifconfig.me`
4. **Создать 100 файлов одной командой:**  
    `touch file{1..100}.txt`

### Полезные ресурсы

- Explainshell — Вставьте любую команду, и сервис объяснит, что делает каждый флаг.
- **Man-страницы:** Всегда пишите `man название_команды` (например, `man awk`), чтобы прочитать документацию прямо в терминале.
