### Настройка Tor поверх Xray/Nekoray (Arch Linux)

Эта конфигурация позволяет обходить жесткие блокировки, упаковывая трафик Tor в обфусцированный туннель (VLESS/Reality) вашего прокси-сервера.

##### 1. Установка необходимых компонентов

Установите основной пакет Tor и плагин для работы мостов `obfs4`:
```bash
sudo pacman -Syu tor obfs4proxy
```

##### 2. Настройка Nekoray

Убедитесь, что в Nekoray включен **Mixed-порт** (SOCKS + HTTP).

- По умолчанию в Nekoray это: `127.0.0.1` порт `2080`.
- Режим **TUN** может быть включен, но для Tor мы будем использовать явное указание прокси.

##### 3. Конфигурация Tor (`/etc/tor/torrc`)

Полностью очистите файл `/etc/tor/torrc` и вставьте следующую конфигурацию:
```bash
# Запуск от системного пользователя
User tor
DataDirectory /var/lib/tor

# Порт, который будет слушать Tor для браузера
SocksPort 127.0.0.1:9050

# Пересылка трафика Tor во входящий порт Nekoray
Socks5Proxy 127.0.0.1:2080

# Включение мостов (обфускация внутри туннеля Xray)
UseBridges 1
ClientTransportPlugin obfs4 exec /usr/bin/obfs4proxy

# Свежие мосты (получить у @GetBridgesBot в Telegram)

Bridge obfs4 162.55.184.210:29299 122E4025413F19FBD991DF41B47ACA8A19111D2D cert=bwCGiLb7NnlIC5BbPtEQU2lq73eMtlJbuHY5XHjLkb+yLKiX3hI3N06+qCImamam74FgTQ iat-mode=0

# Логирование для отладки
Log notice stdout

```

##### 4. Управление сервисом

После правки конфига примените настройки:

1. **Проверка синтаксиса**:
```bash
/usr/bin/tor -f /etc/tor/torrc --verify-config
```

2. **Сброс ошибок и запуск**:
```bash
sudo systemctl reset-failed tor
sudo systemctl enable --now tor
```

3. **Просмотр логов (прогресс до 100%)**:
```bash
journalctl -u tor -f
```

##### 5. Настройка браузера (Firefox)

6. Перейдите в **Settings** -> **Network Settings**.
7. Выберите **Manual proxy configuration**.
8. **SOCKS Host**: `127.0.0.1`, **Port**: `9050`.
9. Выберите **SOCKS v5**.
10. **Важно**: Поставьте галочку **"Proxy DNS when using SOCKS v5"**.

##### 6. Проверка

Откройте check.torproject.org. Вы должны увидеть сообщение о том, что браузер настроен на использование Tor.

---
### Устранение неполадок (Quick Tips)

- **Зависло на 10%**: Проверьте, работает ли Nekoray и активен ли там профиль прокси.
- **Address already in use**: Вы пытаетесь запустить `tor` вручную, пока работает `systemctl` сервис. Сначала сделайте `sudo systemctl stop tor`.
- **TLS_ERROR**: Означает, что трафик Tor детектируется. Убедитесь, что строки `Bridge obfs4` актуальны и плагин `obfs4proxy` прописан верно.

### Что получилил
1. **РФ (Твой ПК)**: Трафик шифруется и маскируется под обычный HTTPS (через VLESS/Reality).
2. **(Твой VPS)**: Здесь трафик «раздевается» от Xray-оболочки. Провайдер в Нидерландах видит, что ты подключаешься к узлам Tor (но так как там это легально, всё ок).
3. **Сеть Tor (Г.)**: Твой трафик проходит еще через 3 узла внутри сети Tor. Г. — это твой **Exit Node** (выходной узел), поэтому сайты видят тебя как нем.

#### Пара советов, чтобы не «спалиться»:

- **WebRTC**: В Firefox вбей в адресную строку `about:config`, найди `media.peerconnection.enabled` и поставь в **false**. Это предотвратит утечку твоего реального IP через браузер.
- **Fingerprinting**: Не разворачивай окно браузера на весь экран — сайты могут вычислить тебя по разрешению монитора (Tor Browser по умолчанию открывается в небольшом окне именно поэтому). 

Теперь ты официально **цифровой призрак**. Если захочешь сменить «страну выхода» с Германии на другую, это можно сделать через параметр `ExitNodes {us}` (например, для США) в том же `torrc`, но обычно Tor сам выбирает лучшие узлы.
