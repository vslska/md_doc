# Инструкция по поиску параллельных сессий в Debian

Если под одной учетной записью на сервере работали одновременно с разных хостов, вычислить второй IP-адрес можно следующими командами.

## 1. Быстрый поиск IP-адресов (Команда `last`)

Выводит список всех входов с принудительным отображением IP-адресов. Ищите одинаковое время и разные IP.

```bash
last -i $(whoami)
```

*   **Где искать:** Третья колонка покажет IP-адрес, с которого было совершено подключение.

---

## 2. Детальный аудит через логи авторизации (`auth.log`)

Позволяет узнать точное время входа, IP-адрес и метод авторизации (пароль или конкретный SSH-ключ).

### Поиск в текущем логе:
```bash
sudo grep "Accepted" /var/log/auth.log | grep "$(whoami)"
```

### Поиск в архивных (сжатых) логах, если событие было давно:
```bash
sudo zgrep "Accepted" /var/log/auth.log* | grep "$(whoami)"
```

**Пример строки из вывода:**
> `Accepted publickey for username from 192.168.1.150 port 51234 ssh2: RSA SHA256:...`  
> *(Здесь `192.168.1.150` — IP-адрес подключившегося хоста)*

---

## 3. Проверка активных сессий (Прямо сейчас)

Если есть подозрение, что второй пользователь все еще находится в системе.

### Кто авторизован и с каких IP:
```bash
w
```

### Дерево процессов пользователя с ID (PID) их терминалов:
```bash
pstree -p $(whoami)
```
```bash
who -p
```
