### Arch-Way: Развертывание локальной ИИ-станции

**Железо:** Ryzen AI 9 HX370 (Zen 5 + NPU XDNA 2)  
**ОС:** Arch Linux  
**Цель:** Полная автономность, работа с кодом и написание ИТ-статей без облаков.

### 1. Подготовка системы (Ядро и Драйверы)

Для архитектуры **Strix Point** (Ryzen AI 9) критически важно свежее ядро. На февраль 2026 года в Arch уже должна быть стабильная поддержка NPU.

```bash
# Установка микрокода AMD
sudo pacman -S amd-ucode

# Рекомендуется использовать ядро linux-mainline для лучшей поддержки Zen 5
yay -S linux-mainline
```
---
### 2. Установка ИИ-движка (Ollama)
**Ollama** — это база для запуска моделей. На Ryzen AI она будет использовать ресурсы iGPU и NPU через библиотеки ROCm.
```bash
# Установка Ollama
sudo pacman -S ollama

# Запуск сервиса
sudo systemctl enable --now ollama

```

#### Выбор моделей для работы:

Для  процессора идеальными будут:
1. **DeepSeek-Coder-V2 (16b):** Для написания кода и конфигов Arch.
2. **Llama-3.3-8B:** Для генерации текстов статей и саммари.
3. **Mistral-Nemo:** Отличная модель для логических задач.
```bash
ollama run deepseek-coder-v2:16b-lite-instruct-q4_K_M
```
---
### 3. Настройка рабочей среды (VSCodium or CodeOSS + Continue)

Вместо платного Cursor мы используем открытый стек, который работает локально.

1. **Установка редактора:**
```bash
yay -S vscodium-bin
```
2. **Установка расширения Continue:**  
Найди в маркетплейсе расширение `Continue`.
3. **Конфигурация `config.json` в Continue:**  
Настрой его на использование локальной Ollama:
```json
{
  "models": [
    {
      "title": "Local DeepSeek (NPU)",
      "provider": "ollama",
      "model": "deepseek-coder-v2"
    }
  ],
  "tabAutocompleteModel": {
    "title": "Starcoder 2b",
    "provider": "ollama",
    "model": "starcoder2:3b"
  }
}
```

#### CODE OSS
##### 1. **Установка и специфика Arch**
```bash
sudo pacman -S code
```
>_Важный нюанс:_ В Code - OSS по умолчанию используется открытый маркетплейс расширений (**Open VSX**), а не проприетарный от Microsoft.

##### 2. **Установка Continue в Code - OSS**

Расширение **Continue** (твой главный инструмент для локального ИИ) есть в Open VSX, так что проблем не будет:
-  Запусти Code.
-  Нажми `Ctrl+P`, введи `ext install continue.continue`.
-  Или просто найди его в поиске по расширениям.

##### 3. **Почему?**

Code - OSS в Arch поставляется с минимальным набором «мусора». Это значит:
- **Меньше фоновых процессов:** Больше ресурсов процессора и NPU уйдет на работу нейросетей (Ollama).
- **Лучшая интеграция с Wayland:** Если ты будешь сидеть на Hyprland или Sway (что на таком железе логично), Code - OSS лучше подхватывает системные флаги.
- _Совет:_ Создай конфиг-файл `~/.config/code-flags.conf` и добавь туда:
```
--ozone-platform-hint=auto
--enable-features=WaylandWindowDecorations
```

##### 4. **Настройка локального «мозга» (Ollama + Code - OSS)**

Будет работать как сервер и клиент одновременно.
-  **В терминале:** Запускаешь модель, оптимизированную под код (например, `deepseek-coder`).
- **В Code - OSS:** В интерфейсе Continue выбираешь локальный провайдер Ollama.

### 4. Интеграция с Arch Wiki (RAG система)
Чтобы ИИ помогал  с Arch Linux максимально точно, используй функцию **Indexing**.

- В Continue нажми `Ctrl+L` и введи `@docs`.
- Выбери `Add new doc`.
- Вставь ссылку: `https://wiki.archlinux.org`.
- **Результат:** Теперь ты можешь спрашивать: _"Как настроить гибернацию на моем железе?"_, и ИИ будет искать ответ именно в актуальной Wiki, а не в своих старых данных.ы
