| Прицип                         | Объяснение                                                                    |
| ------------------------------ | ----------------------------------------------------------------------------- |
| **Разделение ответственности** | HTML = структура, CSS = внешний вид, JS = поведение                           |
| **Статика vs Динамика**        | `static/` — для файлов, которые не меняются; Flask — для динамических страниц |
| **Идемпотентность**            | Каждый элемент имеет **уникальный `id`**, если нужен JS                       |
| **Семантичность**              | Используй `<button>` для кнопок, `<a>` для ссылок — это важно для доступности |
|                                |                                                                               |

Чек лист:

1. Где он в HTML? → `templates/index.html`
2. Как он выглядит? → `static/css/style.css`
3. Как он себя ведёт? → `static/js/main.js`
4. Есть ли у него ресурсы? → `static/img/`, `static/fonts/`

| Слой           | Роль                                           | Пример                                             |
| -------------- | ---------------------------------------------- | -------------------------------------------------- |
| **HTML**       | **Структура** — «кости» страницы               | `<header>`, `<nav>`, `<div>`, `<p>`                |
| **CSS**        | **Макет + стиль** — «кожа и одежда»            | Цвета, отступы, сетка (30/70), шрифты              |
| **JavaScript** | **Поведение + интерактив** — «нервная система» | Анимации, реакция на клик, динамическое обновление |

> [!NOTE] Аналогия
> **HTML** — это **дом**: стены, двери, окна  
**CSS** — это **отделка**: обои, цвет стен, мебель  
**JS** — это **жизнь в доме**: свет включается, дверь хлопает, кофе варится
